## 核心架构演进

### Transformer 基础
GPT 系列基于 **Transformer 解码器架构**，核心组件包括：
- **多头自注意力机制**：捕捉序列中不同位置的关系
- **位置编码**：为 token 提供位置信息
- **前馈神经网络**：逐位置的非线性变换

### 关键创新

**1. 生成式预训练**
- 在大规模无标注文本上，以"预测下一个 token"为目标进行预训练
- 学习语言的统计规律和世界知识

**2. 指令微调（InstructGPT / RLHF）**
- **监督微调（SFT）**：在人工标注的指令数据上微调
- **奖励模型训练**：学习人类偏好排序
- **强化学习（PPO）**：优化策略以符合人类偏好

**3. GPTs 与插件系统**
- 用户可创建自定义 GPT，集成知识库和工具
- 插件生态扩展模型能力边界

## API 使用指南

### 基础调用

```python
from openai import OpenAI

client = OpenAI(api_key="your-key")

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是一个友好的助手。"},
        {"role": "user", "content": "用简单的话解释什么是 Transformer。"}
    ],
    temperature=0.7,
    max_tokens=500
)

print(response.choices[0].message.content)
```

### 常用参数

| 参数 | 作用 | 推荐值 |
|-----|------|-------|
| `temperature` | 控制随机性 (0-2) | 创意类 0.8-1.0，精确类 0-0.3 |
| `top_p` | 核采样阈值 | 0.9 |
| `max_tokens` | 最大输出长度 | 按需设置 |
| `stream` | 流式输出 | True 可提升用户体验 |

## 优势

- **综合能力顶尖**：在推理、编程、写作、翻译等任务上表现均衡
- **生态成熟**：丰富的 API、工具链、社区资源
- **多模态支持**：GPT-4o 支持文本、图像、音频输入
- **函数调用**：支持结构化输出和工具集成
- **插件与 GPTs**：扩展性强

## 局限

- **成本较高**：尤其是 GPT-4 系列 API 调用费用
- **上下文窗口限制**：标准版 128K（GPT-4.1 达 1M）
- **知识截止日期**：训练数据有截止时间
- **幻觉问题**：可能生成看似合理但不正确的内容
- **数据隐私**：通过 API 发送的数据需注意合规

## 应用场景

- **编程助手**：代码生成、调试、重构

- **数据分析**：SQL 生成、数据解读、报告编写
- **教育辅导**：知识问答、题目讲解、学习规划
- **客服系统**：智能对话、工单处理
