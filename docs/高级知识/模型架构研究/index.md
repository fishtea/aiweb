# 模型架构研究

> 理解 Transformer 及其变体是深入 AI 核心的必经之路。从注意力机制到位置编码，从混合专家模型到状态空间模型，本章带你系统梳理 LLM 架构设计的演进脉络。

## 🏛️ Transformer 架构深度解析

标准的 Transformer（如 GPT 系列）由多个相同的层堆叠而成，每一层包含两个核心子层。

### 整体结构

```
输入 Token → Token Embedding → Position Encoding
                                               │
                                               ▼
┌──────────────── 第 L 层 ──────────────────────┐
│  ┌────────────────────────────────────────┐  │
│  │  多头自注意力 (Multi-Head Attention)    │  │
│  │  QKV 投影 → 注意力计算 → 输出投影      │  │
│  └────────────────────────────────────────┘  │
│  │                                           │
│  ┌────────────────────────────────────────┐  │
│  │  Add & LayerNorm（残差连接+层归一化）   │  │
│  └────────────────────────────────────────┘  │
│  │                                           │
│  ┌────────────────────────────────────────┐  │
│  │  FFN (前馈神经网络)                     │  │
│  │  上投影 → 激活 → 下投影               │  │
│  └────────────────────────────────────────┘  │
│  │                                           │
│  ┌────────────────────────────────────────┐  │
│  │  Add & LayerNorm                       │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

### 自注意力（Self-Attention）

注意力机制是 Transformer 的核心创新，其本质是"让每个位置的表示是整个序列的加权组合"。

```
Attention(Q, K, V) = softmax(QK^T / √d_k) × V
```

**计算步骤**：
1. 输入 X 通过线性投影得到 Q, K, V
2. Q 和 K 的点积计算注意力分数
3. 除以 √d_k 缩放（防止 softmax 梯度消失）
4. softmax 归一化为概率分布
5. 加权求和得到输出

### 多头注意力（Multi-Head Attention）

将注意力计算分解为多个"头"，每个头关注不同的表示子空间：

```
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) × W_O

其中 head_i = Attention(Q×W_Q_i, K×W_K_i, V×W_V_i)
```

## 🔄 注意力机制变体

### 多头注意力（MHA）

标准方案，每个头有独立的 Q、K、V 投影。

**参数量**：4 × d_model²（Q、K、V、O 各一个投影矩阵）
**计算复杂度**：O(n² × d)（n 为序列长度）

### 分组查询注意力（GQA）

**创新**：减少 K 和 V 头的数量，Q 头数量不变。多个 Q 头共享一组 K、V 头。

```
MHA:  Q₁ K₁ V₁, Q₂ K₂ V₂, Q₃ K₃ V₃, Q₄ K₄ V₄ (4组)
GQA:  Q₁ Q₂ Q₃ Q₄           (4个Q头)
      K₁ K₂                  (2个K头，被2个Q头共享)
      V₁ V₂                  (2个V头，被2个Q头共享)
```

**优点**：推理时 KV Cache 减少，节省显存。Llama 2/3、Mistral 均采用 GQA。

### 滑动窗口注意力（Sliding Window Attention）

**创新**：每个 token 只关注其周围的 W 个 token，而非全部序列。

```
序列: [A, B, C, D, E, F, G, H]
窗口 (W=3):
D 关注: [B, C, D, E, F]
```

**优点**：线性复杂度 O(n × W)，支持无限长序列
**缺点**：远处信息需要通过多层堆叠传递

### Flash Attention

不改变注意力计算数学，而是通过 IO 感知优化显著加速：

- **核心思想**：在不访问完整注意力矩阵的情况下计算 softmax
- **分块计算**：利用 SRAM（片上高速缓存）分块处理
- **效果**：2-4 倍加速，显存使用从 O(n²) 降到 O(n)

## 🧩 混合专家模型（MoE）

### 基本思想

MoE 将模型拆分为多个"专家"子网络，每个 token 只激活一小部分专家：

```
输入 → Router → [Expert 1] ─┐
             → [Expert 2] ─┤→ 加权求和 → 输出
             → [Expert 3] ─┘
             → [Expert 4] (未选择)
```

### 核心组件

1. **路由器（Router/Gate）**：为每个 token 选择 Top-K 专家
2. **专家（Experts）**：通常是独立的小 FFN 网络
3. **负载均衡**：确保专家被均匀选择，防止某些专家过载

### Mixtral 8x7B

Mistral 的 MoE 模型：
- 8 个专家，每个 token 激活 2 个
- 总参数量 47B，但推理计算量仅相当于 12B
- 效果媲美 Llama 2 70B，但速度快 6 倍

### MoE 的优缺点

| 优点 | 缺点 |
|------|------|
| 等计算量下效果更好 | 路由计算额外开销 |
| 支持更大的总参数量 | 负载均衡训练困难 |
| 推理速度快 | 显存需求高（需加载所有专家） |

## 🌊 状态空间模型（SSM）

### 从 Transformer 到 Mamba

Transformer 的 O(n²) 长序列计算瓶颈催生了替代架构的探索。SSM 将序列建模转化为状态空间方程。

### Mamba

Mamba 是 SSM 的最新变体，核心创新：

**选择性的状态空间模型**：
- 传统的 SSM 是时不变的（参数固定）
- Mamba 的 A、B、C 矩阵变为**输入相关的**
- 这使得模型能像注意力一样"选择性关注"输入

**硬件感知的并行扫描**：
- 使用并行扫描算法高效计算
- 避免注意力机制的 O(n²) 瓶颈
- 实现线性复杂度 O(n)

### Mamba vs Transformer

| 方面 | Transformer | Mamba |
|------|------------|-------|
| 复杂度 | O(n²) | O(n) |
| 长序列 | 显存爆炸 | 线性增长 |
| 训练速度 | 中等 | 快 2-5 倍 |
| 推理速度 | 需要 KV Cache | 状态可压缩 |
| 质量 | 当前最优 | 与 Transformer 持平 |
| 生态 | 成熟 | 新兴 |

## 📍 位置编码

由于自注意力是置换不变的（不考虑 token 顺序），位置编码至关重要。

### 绝对位置编码（Absolute PE）

- 每个位置学习或预设一个唯一的位置向量
- **Sinusoidal PE**：使用正弦/余弦函数生成固定位置编码
- **Learnable PE**：让模型自己学习位置编码

**局限**：无法外推到训练时未见过的序列长度。

### 旋转位置编码（RoPE）

RoPE 是目前最主流的位置编码方法（Llama、Qwen、Mistral 均使用）：

**核心思想**：
- 通过旋转矩阵对 Q 和 K 向量进行变换
- 旋转角度取决于位置差
- 注意力分数自然地反映了相对位置

**优点**：
- 支持相对位置编码
- 可外推到更长序列（通过调整旋转频率）
- 数学优雅，实现简单

### ALiBi（Attention with Linear Biases）

在注意力分数上添加一个基于位置差的线性偏置：

```
Attention Score = QK^T / √d - m × |i - j|
```

**优点**：
- 极简实现，几乎无额外参数
- 天然支持外推
- 训练和推理都非常高效

## 🔤 分词技术

### Tokenization 的本质

将文本分割为模型可处理的离散单元。好的分词器直接影响模型效果。

### 主流方法

| 方法 | 原理 | 代表模型 |
|------|------|---------|
| **BPE** | 从字符逐步合并高频子词对 | GPT, Llama |
| **Unigram** | 基于概率模型的最优分段 | SentencePiece |
| **WordPiece** | BPE 变体，基于似然增益 | BERT |

### 分词中的关键问题

- **词汇表大小**：通常 32K-128K token
- **罕见词处理**：分解为更小的子词单元
- **多语言支持**：需要覆盖多种语言的字符集
- **数字表示**：逐数字分词 vs 整体分词

## 🔮 未来方向

### 混合架构

将 Transformer、MoE、Mamba 的优点结合：

- **Jamba**：Transformer + Mamba 混合层
- **MoE + Attention**：Mixtral 和 GPT-4 采用的方案
- **线性注意力**：用核方法近似注意力计算

### 长期趋势

1. **替代注意力**：寻找比注意力更高效的 token 混合机制
2. **动态计算**：根据任务复杂度动态分配计算资源
3. **硬件协同设计**：架构与硬件（GPU/TPU/NPU）联合优化
4. **持续学习**：模型在部署后持续更新知识
