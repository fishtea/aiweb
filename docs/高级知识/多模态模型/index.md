# 多模态模型

> 人类通过视觉、语言、听觉等多种模态感知世界。多模态 AI 让机器拥有了同样的能力——从理解图像和语音，到生成逼真的图片和视频，AI 正在突破文本的边界。

## 🎯 什么是多模态 AI？

多模态 AI 是指能够处理和理解**多种类型数据**的 AI 系统。常见模态包括：

| 模态 | 输入 | 输出 | 代表模型 |
|------|------|------|---------|
| **文本** | 文字 | 文字 | GPT-4, Llama |
| **图像** | 图片 | 图片/描述 | CLIP, DALL-E |
| **语音** | 音频 | 文本/音频 | Whisper, AudioCraft |
| **视频** | 视频帧 | 描述/视频 | VideoPoet |
| **代码** | 代码 | 代码/解释 | CodeLlama |

### 单模态 vs 多模态

```
单模态（文本到文本）：
用户输入：[文本] "描述这张图片"
AI 输出：[文本] "无法看到图片"

多模态（图像+文本到文本）：
用户输入：[图片] 🖼️ + [文本] "描述这张图片"
AI 输出：[文本] "这是一张日落时分的海滩照片..."
```

## 👁️ 视觉-语言模型

### CLIP（Contrastive Language-Image Pre-training）

OpenAI 开创性的多模态模型，核心思想是对比学习：

```
                  [文本编码器] → 文本特征向量
图片 → [图像编码器] → 图片特征向量 → 对比学习 → 相似度矩阵
```

**工作原理**：
- 将图片和描述文本分别编码到同一向量空间
- 通过对比损失，让匹配的图片-文本对距离更近
- 不匹配的图片-文本对距离更远

**应用场景**：
- **零样本分类**：无需训练，直接通过文本描述识别图片内容
- **图文检索**：用文本搜索图片，或用图片搜索相关文本
- **多模态理解**：作为多模态模型的视觉 backbone

### LLaVA（Large Language and Vision Assistant）

将视觉编码器与 LLM 连接的多模态框架：

```
图像 → CLIP 视觉编码器 → 视觉特征 → 投影层 → LLM → 输出
文本 → 文本嵌入 ──────────────────────────↑
```

LLaVA 的创新点：
- 使用简单的线性投影层连接视觉和语言
- 训练分为两阶段：先预训练投影层，再端到端微调
- 在多种视觉-语言任务上表现优异

### GPT-4V / GPT-4o

目前最强大的商用多模态模型：
- 支持图片输入、分析和推理
- 能识别图表、公式、手写文字
- 理解图像中的空间关系
- GPT-4o 进一步支持音频实时交互

## 🎨 图像生成

### 扩散模型（Diffusion Models）

当前的图像生成核心技术，从随机噪声逐步"去噪"生成图片：

```
纯噪声 → 第 T 步 → ... → 第 t 步 → ... → 第 0 步 → 最终图像
  🎲      模糊            稍清晰            清晰
```

**前向过程**：逐步向图片添加噪声，直到变为纯噪声
**反向过程**：训练神经网络预测并去除噪声

### Stable Diffusion

在潜在空间中进行扩散（Latent Diffusion），大幅降低计算成本：

- 使用 VAE 将图像压缩到低维潜在空间
- 在潜在空间中进行扩散和去噪
- 使用 CLIP 文本编码器作为条件控制

```python
# Stable Diffusion 推理示例
from diffusers import StableDiffusionPipeline

pipe = StableDiffusionPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5"
)
image = pipe(
    prompt="一只穿着宇航服的猫，在火星上漫步",
    negative_prompt="模糊，低质量",
    num_inference_steps=50,
    guidance_scale=7.5
).images[0]
```

**关键概念**：
| 概念 | 说明 |
|------|------|
| **Guidance Scale** | 控制文本条件强度，越大越遵循提示但可能失真 |
| **CFG（Classifier-Free Guidance）** | 混合条件和非条件预测 |
| **Sampler** | 去噪调度器（DDIM, DPM++等） |
| **LoRA** | 微调扩散模型生成特定风格 |
| **ControlNet** | 通过姿态、深度图等精确控制生成 |

### DALL-E 3

OpenAI 的图像生成模型，支持多轮对话式图像生成：
- 使用 GPT-4 优化用户输入为详细描述
- 生成与描述高度匹配的图像
- 支持图片局部编辑和重绘

## 🎙️ 语音与音频

### Whisper

OpenAI 开源的通用语音识别模型：

- 支持 99+ 种语言
- 多任务：语音转文字、翻译、语言识别
- 鲁棒性强：对噪声、口音、长音频都表现良好
- 有多个大小版本（tiny, base, small, medium, large）

```python
import whisper

model = whisper.load_model("medium")
result = model.transcribe("演讲录音.mp3")
print(result["text"])  # 转录文本
```

### AudioCraft

Meta 开源的音频生成套件：
- **MusicGen**：文本生成音乐
- **AudioGen**：文本生成音效
- **EnCodec**：神经音频压缩

## 🏗️ 统一多模态架构

### 当前趋势：一个模型处理所有模态

| 模型 | 模态 | 架构 |
|------|------|------|
| **GPT-4o** | 文本+图像+音频 | 统一 transformer |
| **Gemini** | 文本+图像+音频+视频 | 多模态 transformer |
| **ImageBind** | 文本+图像+音频+深度+热感+IMU | 联合嵌入空间 |
| **Any-to-Any** | 多种输入和输出 | Encoder-Decoder |

### Meta 的 ImageBind

创新点：不需要所有模态对的训练数据，只需要**图像作为"绑定"中心**：

```
文本 ─┐
图像 ─┤
音频 ─┤→ 同一个嵌入空间
深度 ─┤
热感 ─┘
```

只要有图像+文本、图像+音频、图像+深度等成对数据，就能将所有模态映射到同一空间。

## 🛠️ 训练多模态模型

### 训练策略

1. **预训练阶段**：使用大规模图文对进行对比学习
2. **对齐训练**：训练投影层将不同模态特征对齐
3. **指令微调**：在多模态指令数据上微调

### 训练数据

| 数据类型 | 来源 | 用途 |
|---------|------|------|
| 图文对 | LAION-5B, CC12M | 预训练 |
| 图像描述 | COCO, Flickr30k | 对齐训练 |
| 视觉问答 | VQA v2, VizWiz | 指令微调 |
| 多轮对话 | LLaVA-Instruct | 对话能力 |

### 硬件需求

| 模型 | 训练硬件 | 推理硬件 |
|------|---------|---------|
| CLIP | 32×V100 | 1×V100 |
| LLaVA-7B | 8×A100 | 1×A100 |
| Stable Diffusion | 8×A100 | 1×RTX 4090 |
| GPT-4V 级别 | 千卡级集群 | API 调用 |

## 🚀 应用与未来方向

### 当前应用

- **文档理解**：扫描文档、识别表格、提取信息
- **内容创作**：文生图、图生文、视频生成
- **辅助视觉**：为视障人士描述场景
- **教育**：交互式学习、公式识别、解题指导
- **医疗**：医学影像分析、报告生成

### 未来方向

1. **视频理解**：从静态图像扩展到动态视频理解
2. **3D 理解**：三维场景的生成和理解
3. **具身智能**：多模态 + 机器人控制
4. **世界模型**：通过多模态输入构建对物理世界的理解
5. **实时交互**：低延迟的多模态对话
