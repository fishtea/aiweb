# 数据工程与合成数据

模型能力很大程度上受数据质量影响。无论是预训练、微调、RAG 还是评估，数据工程都决定了系统上限，合成数据则是补齐稀缺场景、构造评估集和提升指令覆盖的重要手段。

## 数据在 AI 系统中的位置

| 场景 | 数据目标 |
|------|----------|
| 预训练 | 覆盖广泛知识、语言模式、代码和多模态内容 |
| 指令微调 | 教会模型遵循任务、格式、语气和安全边界 |
| 偏好优化 | 用人类或模型偏好样本塑造回答风格和对齐表现 |
| RAG | 提供可追溯、可更新、带权限的外部知识 |
| 评估 | 用代表性样例衡量质量、鲁棒性、安全性和回归风险 |

## 数据治理流程

1. 明确数据用途：训练、检索、评估、监控或反馈。
2. 定义数据 schema：字段、来源、时间、权限、标签和质量评分。
3. 清洗去重：删除重复、乱码、广告、页眉页脚、低质量模板文本。
4. 分层抽样：覆盖主任务、长尾任务、失败案例和高风险场景。
5. 标注与审核：建立标注指南、冲突处理和一致性检查。
6. 版本管理：记录数据集版本、构建脚本、过滤规则和变更说明。
7. 闭环迭代：把线上失败案例转化为评估集、训练集或知识库修订。

## 合成数据适用场景

- 真实样本稀缺，但任务规则明确。
- 需要覆盖边界条件、异常输入和安全对抗样例。
- 需要把专家知识转成问答、步骤、分类或工具调用样本。
- 需要扩展评估集，验证 Prompt、RAG 或 Agent 工作流是否回归。

## 合成数据风险

| 风险 | 处理方式 |
|------|----------|
| 幻觉扩散 | 合成前提供可靠来源，合成后做事实校验 |
| 分布偏移 | 混合真实样本，定期对比线上输入分布 |
| 格式单一 | 使用多种模板、难度、语气和错误类型 |
| 标签污染 | 分离训练集、验证集和测试集，避免同源泄漏 |
| 安全问题 | 对敏感、违法、隐私内容做过滤和审查 |

## 高质量样本特征

- 输入真实，包含业务上下文，而不是过度理想化问题。
- 输出可验证，最好有引用、规则、计算过程或人工标准答案。
- 难度分层，覆盖简单、普通、困难和拒答场景。
- 标签清晰，能说明为什么这个答案是好的或差的。
- 可追溯，知道样本来自真实用户、专家编写还是模型合成。

## 2026 最新进展：合成数据与训练方法演进

### 知识蒸馏在 LLM 时代的复兴

知识蒸馏（Knowledge Distillation, KD）并非新技术，但在 2025-2026 年 LLM 时代获得了新的生命力。根据 HuggingFace 社区的综述（Kseniase, 2025），现代 KD 已经从简单的"大模型教小模型"演化为系统化的方法论。

**核心概念**：KD 遵循"幂律关系"（power law）——性能随资源投入可预测地提升，但达到一定点后，继续增加资源不再带来显著改善。这使 KD 成为在有限资源下最大化模型能力的关键技术。

**主要蒸馏方法演进**：

| 方法 | 原理 | 适用场景 |
|------|------|----------|
| 响应蒸馏（Response-based） | 让小模型模仿大模型的输出分布 | 通用场景，最简单 |
| 特征蒸馏（Feature-based） | 对齐中间层表示 | 同架构模型间迁移 |
| 关系蒸馏（Relation-based） | 保持样本间的关系结构 | 需要保持细粒度判别能力 |
| 在线蒸馏（Online KD） | 师生模型同时训练，互相学习 | 无预训练大模型可用时 |

**DeepSeek 案例**：DeepSeek 通过知识蒸馏将大模型能力迁移至小模型取得了显著成果，但也引发了关于蒸馏边界和许可合规的争议——这凸显了蒸馏技术在实践中的潜力和挑战。

> 参考来源：Kseniase, "Everything You Need to Know about Knowledge Distillation", HuggingFace Blog, 2025-03。

### RL Post-Training：PPO / DPO / GRPO 深入对比

2026 年强化学习后训练（RL Post-Training）已成为 LLM 训练流程的标准步骤。根据 HuggingFace 社区的详细指南（karina-zadorozhny, 2026-01），主流方法分为 On-Policy 和 Off-Policy 两大类：

**On-Policy 方法（模型在训练中自己生成数据）**：

- **PPO（Proximal Policy Optimization）**：经典 RLHF 主力算法。通过截断重要性采样比率（clipped surrogate objective）限制策略更新幅度，防止训练不稳定。PPO-CLIP 是实践中最广泛采用的变体，PPO-KLPEN 通过显式 KL 惩罚提供更软性的约束。

- **GRPO（Group Relative Policy Optimization）**：PPO 的简化变体，通过对同组响应的相对比较来估计优势函数，减少了对价值网络（value network）的依赖。

**Off-Policy 方法（使用静态偏好数据集）**：

- **DPO（Direct Preference Optimization）**：通过巧妙的数学变换，将偏好优化转化为直接的分类问题，无需显式奖励模型。DPO 内存效率高且训练稳定，但对数据噪声敏感，容易过拟合。

- **迭代 DPO（Iterative DPO）**：解决静态数据集分布偏移问题，通过多轮"生成→标注→训练"循环逐步改善模型。

**KL 散度的关键作用**：RL Post-Training 中的 KL 惩罚起到防止模型偏离原始能力太远的作用。文章指出，实际使用的是"漂移 KL（Drift KL）"而非"奖励 KL"，且 Reverse KL（mode-seeking）比 Forward KL（mean-seeking）更适合此场景——它允许模型忘记不需要的模式（如有毒回复），同时保持核心能力。

> 参考来源：karina-zadorozhny, "A Guide to Reinforcement Learning Post-Training for LLMs: PPO, DPO, GRPO, and Beyond", HuggingFace Blog, 2026-01-19。

### 合成数据的质量管控升级

基于 2024-2025 的实践教训，2026 年合成数据的质量管控有了更明确的标准：

1. **分布验证**：不再是简单的"人工抽检"，而是通过聚类和降维可视化（t-SNE / UMAP）验证合成数据覆盖了真实数据的分布空间。
2. **难度分层自动化**：使用模型置信度、生成长度、推理步数等信号自动标注样本难度，确保训练集包含从基础到挑战的完整梯度。
3. **去污与去重强化**：在大规模合成场景中，MinHash 和 SimHash 等近似去重算法成为标配，训练集和测试集间的语义去重（而非仅字符串匹配）被普遍采用。
4. **闭环反馈**：将生产环境中模型的失败案例（用户点踩、人工纠错）回流到数据管线，自动生成针对性的合成训练样本。

## 2026 最新实践：数据质量与工具链升级

### 数据版本控制成为"新常态"

2025-2026 年，数据版本控制从"最佳实践建议"升级为"工程刚需"。随着模型迭代加速和监管要求趋严，三大变化正在发生：

1. **数据血缘追踪（Data Lineage）**：不仅要记录"这个数据集是什么版本"，还要回答"它从哪来、经过了什么处理、被哪些模型用过"。类似 DVC（Data Version Control）和 Pachyderm 的工具正在从实验走向生产。

2. **差分审计（Differential Auditing）**：新趋势是不仅比较"当前数据集 vs 上一版本"，还通过嵌入空间聚类（embedding clustering）检测数据集漂移——是否新增了某种偏见、是否某类话题被过度代表。

3. **数据卡片（Data Cards）标准化**：类似于模型的"模型卡片"（Model Cards），数据集开始附带标准化的数据卡片——记录数据来源、采集方式、已知偏见、适用场景和限制。

### 合成数据的"可信度"衡量框架

随着合成数据在训练中的占比不断提高，业界开始建立衡量合成数据可信度的框架：

| 维度 | 关键问题 | 评估方法 |
|------|---------|----------|
| 分布一致性 | 合成数据是否覆盖真实数据的分布空间？ | t-SNE/UMAP 可视化 + Wasserstein 距离 |
| 因果保真度 | 合成样本的输入-输出关系是否符合真实因果逻辑？ | 人工标注子集 + 反事实测试 |
| 多样性与冗余 | 是否存在重复模式或模板化答案？ | MinHash 聚类 + 自我 BLEU |
| 安全性 | 合成过程是否引入新的偏见或毒性？ | 偏见评测基准 + 红队探测 |
| 可追溯性 | 能否区分合成样本和真实样本？ | 水印嵌入 + 元数据标记 |

> **行业趋势：** 越来越多的团队采用"合成-验证-混合"三段式管线：先用大模型生成合成数据 → 用规则/小模型做快速质量过滤 → 按比例混合真实样本和高质量合成样本。关键是**合成数据永远应该与真实数据混合使用**而非完全替代。
