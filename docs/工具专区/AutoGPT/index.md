# AutoGPT

> AutoGPT 是一个基于大语言模型的自主 AI 代理系统，能够自动分解目标、制定计划并使用工具一步步完成任务。

---

## 什么是 AutoGPT？

AutoGPT 是 AI 代理概念的标志性实现。与传统 AI 助手（用户提问→AI 回答）不同，AutoGPT 可以：

- **自主设定子目标**：将大目标分解为可执行的步骤
- **循环执行**：执行→观察结果→调整计划→继续执行
- **使用工具**：调用搜索引擎、文件系统、代码执行器等
- **长期记忆**：记忆上下文和学习经验

---

## 核心架构

```
用户输入目标
     ↓
[任务管理器] → 分解为子任务
     ↓
[执行循环] → 选择工具 → 执行 → 观察结果 → 评估
     ↓                     ↕
[记忆系统] → 短期记忆（上下文）
           → 长期记忆（向量数据库）
           → 文件存储
```

### 执行循环

AutoGPT 的核心是持续的执行循环：

1. **思考（Thought）**：分析当前状态
2. **推理（Reasoning）**：决定下一步行动
3. **计划（Plan）**：制定执行方案
4. **批评（Criticism）**：自我审查方案的合理性
5. **执行（Action）**：调用工具完成任务
6. **观察（Observation）**：获取执行结果
7. **重复（Repeat）**：直到目标完成

## 快速开始

### 安装与运行

```bash
# 克隆仓库
git clone https://github.com/Significant-Gravitas/AutoGPT.git
cd AutoGPT

# 配置
cp .env.example .env
# 编辑 .env 填入 OPENAI_API_KEY

# Docker 运行
docker compose build
docker compose up
```

### 配置工具


## 记忆系统

### 三种记忆层级

| 类型 | 存储方式 | 特性 | 用途 |
|-----|---------|------|------|
| **上下文记忆** | 对话窗口 | 短期、有限 | 当前执行上下文 |
| **向量记忆** | 向量数据库 | 长期、可检索 | 重要信息和经验 |
| **文件记忆** | 文件系统 | 永久 | 生成的产物和日志 |

### 向量记忆

```python
# AutoGPT 使用嵌入进行记忆检索
from autogpt.memory.vector import VectorMemory

memory = VectorMemory()
memory.add("用户在讨论 Python 装饰器", metadata={"topic": "python"})
results = memory.search("python 相关讨论", k=5)
```

## 与 LangChain Agent 的对比

| 维度 | AutoGPT | LangChain Agent |
|-----|---------|----------------|
| **目标** | 自主完成任务 | 辅助完成单次查询 |
| **流程控制** | 自动循环，持续执行 | 单轮或多轮对话 |
| **自主性** | 高（自主分解目标） | 中（需提示引导） |
| **复杂度** | 高 | 中等 |
| **可控性** | 低（自主决策多） | 高（每一步可控） |
| **适用场景** | 长时间自主任务 | 交互式应用 |
| **失败风险** | 较高（可能跑偏） | 较低 |

## 优势

- **全自主执行**：设定目标后无需人工干预
- **目标分解**：自动将复杂任务分解为可执行步骤
- **工具丰富**：内置搜索、代码执行等实用工具
- **长期记忆**：跨会话保持经验和知识
- **开源可定制**：可自行扩展工具和行为逻辑

## 局限

- **成本较高**：持续循环调用 API，token 消耗大
- **可靠性问题**：可能陷入循环或偏离目标
- **速度慢**：每一步都需要 LLM 推理，整体耗时较长
- **调试困难**：自主决策过程难以完全追踪
- **安全风险**：自主执行代码和操作需谨慎

## 实际应用案例

### ✅ 适合的任务

- **市场研究**：搜索信息 → 整理分析 → 生成报告
- **代码开发**：分析需求 → 编写代码 → 测试调试
- **数据收集**：爬取网页 → 提取数据 → 结构化存储
- **内容创作**：研究主题 → 撰写文章 → 优化润色

### ❌ 不适合的任务

- **实时交互**：需要即时响应的对话
- **高精度任务**：结果必须完全准确的场景
- **敏感操作**：涉及资金、权限的操作

---

## 最佳实践

1. **目标明确**：设定清晰、具体的最终目标
2. **分步推进**：大目标拆分为多个 AutoGPT 任务
3. **设置约束**：添加安全限制和操作边界
4. **定期检查**：观察执行进度，及时干预
5. **从简单开始**：先在小任务上验证
