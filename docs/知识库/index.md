# 知识库使用指南

这套文档按“学习路径 + 专题知识 + 工程实践”的方式组织，目标不是简单堆积链接，而是把 AI 学习中常见的概念、模型、工具和实战流程整理成可检索、可复用、可持续更新的中文知识库。

## 内容结构

| 模块 | 解决的问题 | 适合人群 |
|------|------------|----------|
| [初级知识](/初级知识/) | 建立 AI、机器学习、深度学习和大语言模型的基础概念 | 零基础学习者、产品和业务同学 |
| [进阶学习](/进阶学习/) | 掌握提示词、RAG、Agent、微调和评估等应用能力 | AI 应用开发者、技术负责人 |
| [高级知识](/高级知识/) | 理解训练优化、安全对齐、多模态和模型架构演进 | 算法工程师、研究型读者 |
| [模型专区](/模型专区/) | 对比 GPT、Claude、LLaMA、DeepSeek、Gemini、Qwen 等模型生态 | 选型、集成和模型评估场景 |
| [工具专区](/工具专区/) | 查找 LangChain、vLLM、Ollama、PyTorch、TensorFlow 等工具用法 | 工程落地、原型验证、部署优化 |
| [AI Agent 实践](/AIAgent实践/) | 将函数调用、RAG、多 Agent 协作落到可复用系统设计中 | Agent 开发者、自动化流程设计者 |

## 推荐学习路径

### 入门路径

1. 从 [人工智能入门](/初级知识/人工智能入门/) 理解 AI 的基本边界、能力来源和应用场景。
2. 阅读 [机器学习基础](/初级知识/机器学习基础/) 和 [深度学习入门](/初级知识/深度学习入门/)，补齐模型训练、数据、损失函数和神经网络概念。
3. 进入 [大语言模型基础](/初级知识/大语言模型基础/)，理解 Transformer、上下文窗口、生成式模型和提示词交互。

### 应用开发路径

1. 先学习 [提示词工程](/进阶学习/提示词工程/)，掌握任务拆解、上下文组织和输出约束。
2. 通过 [Embedding 与向量数据库](/进阶学习/Embedding与向量数据库/) 和 [RAG 检索增强](/进阶学习/RAG检索增强/) 学会把私有知识库接入大模型。
3. 阅读 [Agent 智能体](/进阶学习/Agent智能体/) 与 [函数调用 Agent](/AIAgent实践/函数调用Agent/)，把模型输出连接到工具、API 和业务流程。
4. 使用 [模型评估与基准](/进阶学习/模型评估与基准/) 与 [Agent 评估与可观测性](/AIAgent实践/Agent评估与可观测性/) 建立上线前后的质量评估闭环。

### 工程部署路径

1. 先读 [LLM 应用架构](/进阶学习/LLM应用架构/)，明确应用分层、质量闭环和运维边界。
2. 在 [模型专区](/模型专区/) 和 [开源模型部署选型](/模型专区/开源模型部署选型/) 对比候选模型的能力、成本、部署方式和生态支持。
3. 结合 [工具专区](/工具专区/) 与 [部署运维](/工具专区/部署运维/) 选择推理、编排、向量检索、本地运行和训练框架。
4. 深入 [模型训练与优化](/高级知识/模型训练与优化/) 和 [模型微调技术](/进阶学习/模型微调技术/)，判断何时需要微调、量化、蒸馏或分布式训练。

## 知识入库标准

新增内容建议按以下标准整理，保证后续检索和维护成本可控：

| 维度 | 要求 |
|------|------|
| 主题归属 | 每篇文档只解决一个清晰主题，放入最贴近的一级栏目和专题目录 |
| 读者定位 | 开头说明适合谁、需要什么前置知识、读完能解决什么问题 |
| 结构完整 | 优先包含概念解释、核心机制、实践步骤、常见误区、适用场景和延伸阅读 |
| 来源可追溯 | 对模型、工具、框架和论文相关内容保留官方文档、论文或可信资料链接 |
| 可操作性 | 工程类内容应给出环境假设、关键命令、配置示例、验证方法和排错线索 |
| 更新策略 | 涉及模型版本、工具 API、价格、排行榜等易变化信息时，需要标注更新时间 |

## 推荐文档模板

```md
# 主题名称

一句话说明本文解决什么问题。

## 适合读者

- 读者画像
- 前置知识
- 学完后的能力

## 核心概念

解释关键术语、工作机制和常见误区。

## 实践步骤

1. 环境准备
2. 最小可运行示例
3. 参数和配置说明
4. 验证结果

## 选型建议

说明适合场景、不适合场景、成本和风险。

## 延伸阅读

- 官方文档
- 论文或技术报告
- 高质量教程
```

## 维护清单

- 每次新增专题后，检查 `docs/.vitepress/config.mts` 是否需要补充导航和侧边栏。
- 每次新增模型或工具页后，检查对应栏目页是否需要更新表格入口。
- 对自动采集生成的内容，优先补充摘要、术语解释、实践步骤和来源说明。
- 对已经过时的工具命令、模型名称、API 示例和排行榜信息，保留更新时间并及时修订。
- 构建前运行 `npm run build`，确认链接、侧边栏和静态站点输出正常。
