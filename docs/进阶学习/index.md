# 🔧 进阶学习

掌握 AI 核心实践技能，学习模型调优与系统构建。

本栏目连接基础概念和工程落地：先用提示词把模型能力稳定释放出来，再用 RAG 接入私有知识，用 Agent 调用工具和执行流程，最后通过微调与评估建立质量闭环。

| 主题 | 简介 |
|------|------|
| [模型微调技术](模型微调技术/index.md) | LoRA、QLoRA、PEFT 等微调技术 |
| [RAG 检索增强](RAG检索增强/index.md) | 检索增强生成，结合知识库提升 LLM 准确性 |

> 💡 建议顺序：提示词工程 → Embedding 与向量数据库 → RAG → Agent → LLM 应用架构 → 微调 → 评估

## 常见目标

- 构建企业知识库问答：先学习 [Embedding 与向量数据库](Embedding与向量数据库/index.md)，再进入 [RAG 检索增强](RAG检索增强/index.md) 和 [模型评估与基准](模型评估与基准/index.md)。
- 构建自动化助手：优先学习 [Agent 智能体](Agent智能体/index.md)，再进入 [AI Agent 实践](/AIAgent实践/)。
- 设计生产系统：阅读 [LLM 应用架构](LLM应用架构/index.md)，再补齐部署、监控、评估和安全策略。
- 提升特定任务效果：先确认提示词、检索和数据是否足够，再评估是否需要 [模型微调技术](模型微调技术/index.md)。

## 学习检查点

- 能把一个业务问题拆成提示词、检索、工具调用、评估和运维几个部分。
- 能判断什么时候用 RAG，什么时候用微调，什么时候只需要改提示词。
- 能为 LLM 应用设计最小评估集，并持续跟踪答案质量、成本和延迟。
- 能解释 Agent 失败时应先检查任务边界、工具权限、上下文质量还是模型能力。
