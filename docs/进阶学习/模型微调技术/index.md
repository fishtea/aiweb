# 模型微调技术

> 将通用预训练模型转化为领域专家的核心方法。微调（Fine-tuning）是当前最实用的 LLM 定制技术。

## 🤔 为什么要微调？

预训练模型（如 Llama、Qwen、Mistral）在通用任务上表现优异，但在特定场景下往往不够精准。微调解决了以下问题：

| 需求 | 通用模型的问题 | 微调后的效果 |
|------|---------------|-------------|
| **领域适应** | 缺乏专业术语和领域知识 | 准确理解医学/法律/金融等专业内容 |
| **任务特定** | 输出格式不符合要求 | 按指定格式生成结构化输出 |
| **风格控制** | 语气、风格不可控 | 输出风格统一、符合品牌调性 |
| **知识更新** | 知识截止于训练数据 | 注入最新知识或私有数据 |

## 🔬 全量微调 vs 参数高效微调

### 全量微调（Full Fine-Tuning）

更新模型**所有参数**，计算成本高，需要多 GPU。

**优点**：理论上能达到最好效果
**缺点**：每一任务需要一份完整模型副本，存储成本高

### 参数高效微调（PEFT）

只更新少量参数，冻结大部分预训练权重。主流方法：

| 方法 | 参数更新量 | 显存需求 | 效果 |
|------|-----------|---------|------|
| **LoRA** | 0.1% - 1% | 低 | 接近全量微调 |
| **QLoRA** | 0.1% - 1% | 极低 | 略低于 LoRA |
| **Adapter** | 1% - 5% | 中等 | 不错 |
| **Prefix Tuning** | 0.1% | 低 | 任务相关 |

### 📌 LoRA（Low-Rank Adaptation）

LoRA 的核心思想：权重更新矩阵 ΔW 是低秩的，可分解为两个小矩阵 A 和 B 的乘积。

```
原始推理：y = Wx
LoRA 推理：y = Wx + BAx   (B∈ℝ^(d×r), A∈ℝ^(r×d), r << d)
```

- **r（秩）**：通常设为 8-64，控制参数量
- **alpha**：缩放因子，通常为 r 的 2 倍
- **应用位置**：通常加在注意力层的 Q、K、V、O 投影上

### 📌 QLoRA

QLoRA = LoRA + 4-bit 量化，在显存受限时非常实用：

- 将预训练权重量化到 4-bit NF4 格式
- 只对量化后的低精度权重做 LoRA 微调
- 可在单张 24GB 显卡上微调 70B 模型

## 📊 数据准备

数据质量直接决定微调效果。

### 数据格式

最常用的是 **对话格式**（ChatML 风格）：

```json
{
  "messages": [
    {"role": "system", "content": "你是一个专业的医疗助手。"},
    {"role": "user", "content": "头痛应该怎么办？"},
    {"role": "assistant", "content": "头痛的原因很多，建议先休息观察..."}
  ]
}
```

### 数据质量要求

1. **多样性**：覆盖各种输入场景
2. **准确性**：答案必须正确，错误答案会教坏模型
3. **一致性**：格式、风格、术语统一
4. **去重**：高重复率会导致灾难性遗忘
5. **数据量**：1,000 - 10,000 条高质量数据通常就足够

### 数据增强技巧

- 使用 GPT-4 生成种子数据的变体
- 人工标注 + 自动校验
- 反向翻译增强多语言能力

## ⚙️ 训练配置

### 硬件需求参考

| 模型大小 | 全量微调 | LoRA | QLoRA |
|---------|---------|------|-------|
| 7B | 4×A100(80G) | 1×A100(40G) | 1×RTX 4090 |
| 13B | 8×A100(80G) | 2×A100(40G) | 1×A100(40G) |
| 70B | 32×A100(80G) | 4×A100(80G) | 1×A100(80G) |

### 主流框架

- **LlamaFactory**：最易用的微调框架，支持多种模型和 PEFT 方法
- **Hugging Face TRL**：官方支持的强化学习微调工具
- **Axolotl**：灵活的微调配置方案
- **Unsloth**：针对 LoRA 优化的快速训练

## 📝 微调工作流程

```
1️⃣ 数据准备
   ↓
2️⃣ 模型选择 → 选择基座模型
   ↓
3️⃣ 配置参数 → 学习率、批次大小、LoRA 配置
   ↓
4️⃣ 训练监控 → loss、gradient norm、eval loss
   ↓
5️⃣ 评估 → 在测试集上评估
   ↓
6️⃣ 导出部署 → 合并权重或加载 adapter
```

### 关键训练参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 学习率 | 2e-4 (LoRA), 1e-5 (全量) | PEFT 通常用更高的学习率 |
| 批次大小 | 4-128 | 根据显存调整 |
| Epoch | 2-5 | 小数据集更多 epoch |
| 最大序列长度 | 2048-4096 | 影响显存占用 |

## ⚠️ 常见问题与解决方案

### 灾难性遗忘
模型忘记了之前学到的通用知识。

**解决**：混合通用数据 + 任务数据，使用较小的学习率，不要过度训练

### 过拟合
在训练集上表现好但泛化差。

**解决**：增加数据量、使用 dropout、早停（early stopping）、数据增强

### 训练不稳定
Loss 剧烈波动。

**解决**：降低学习率、增加 warmup 步数、检查数据是否有噪声

### 输出质量下降
微调后反而变笨了。

**解决**：检查数据质量、减少微调数据中的"简单"样本、加入高质量通用数据

## 🚀 实践建议

1. **从 LoRA 开始**：先用 LoRA 快速验证数据效果
2. **数据先行**：花 80% 时间在数据准备上
3. **渐进式训练**：先小 batch 小学习率试跑，确认没问题再全速训练
4. **基线对比**：在微调前先用原始模型跑测试，作为对比基线
