# 提示词入门

提示词是你和大语言模型协作的接口。好的提示词不是“咒语”，而是把任务、背景、约束、输出格式和判断标准说清楚。

## 提示词的基本结构

```text
角色：你希望模型以什么身份处理任务
任务：要完成什么
背景：必要上下文和资料
约束：不能做什么、需要注意什么
格式：输出结构
标准：什么算好答案
```

示例：

```text
你是一名中文技术编辑。
请把下面这段 AI 入门内容改写成适合零基础读者阅读的版本。
要求：保留关键术语；避免夸张宣传；用表格解释难点。
输出格式：先给 3 条要点，再给改写正文。
```

## 常用技巧

| 技巧 | 作用 |
|------|------|
| 明确任务 | 避免模型猜测你的目的 |
| 提供背景 | 减少错误假设 |
| 限定范围 | 防止输出发散 |
| 指定格式 | 便于后续复制、解析或入库 |
| 给示例 | 让模型模仿你想要的风格 |
| 要求自检 | 提醒模型检查遗漏和矛盾 |

## 输出格式控制

如果输出要进入系统或表格，应明确格式：

```text
请只输出 JSON，不要解释。
字段包括：
- title：标题
- summary：100 字以内摘要
- tags：不超过 5 个标签
```

如果是给人阅读，可以要求：

- 使用二级标题。
- 每段不超过 120 字。
- 用表格对比方案。
- 最后给出风险和下一步。

## 提示词不适合解决什么

提示词不是万能的。以下问题通常需要其他方法：

| 问题 | 更合适的方法 |
|------|--------------|
| 模型不知道私有资料 | RAG 或上传上下文 |
| 需要稳定结构化输出 | JSON schema、函数调用、后处理校验 |
| 需要长期记忆 | 数据库或用户画像系统 |
| 领域术语大量错误 | 检索增强、少量样例、微调 |
| 输出质量难判断 | 建立评估集和评分标准 |

## 常见错误

- 只写“帮我优化一下”，没有说明目标读者和标准。
- 一次要求太多任务，导致输出不稳定。
- 没有给背景，让模型自行补充事实。
- 要求模型回答最新信息，但没有提供来源。
- 让模型输出结构化数据，却没有做格式校验。

## 入门练习

1. 让模型把一篇文章总结成 5 条要点。
2. 让模型用表格比较两个方案。
3. 给模型一段错误答案，要求它指出问题。
4. 提供一个示例，让模型按同样格式生成。
5. 要求模型输出 JSON，并检查是否能被程序解析。

### 常见采样参数入门

提示词之外，采样参数也直接影响输出。初学者至少理解这几个：

| 参数 | 作用 | 调整直觉 |
|------|------|----------|
| temperature | 控制随机性 | 越高越发散（创意），越低越确定（事实） |
| top_p | 限制采样候选范围 | 和 temperature 二选一调整即可 |
| max_tokens | 限制输出长度 | 防止跑题和成本失控 |
| seed | 固定随机性 | 需要可复现结果时使用 |

> 经验：事实问答、代码、结构化抽取用低 temperature（0-0.3）；创意写作、头脑风暴用高 temperature（0.7-1.0）。没有"万能参数"，要在评估集上对比。
