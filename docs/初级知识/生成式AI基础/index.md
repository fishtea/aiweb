# 生成式 AI 基础

生成式 AI 指能够生成新内容的 AI 系统，包括文本、图片、音频、视频、代码和 3D 内容。大语言模型和图像扩散模型是当前最常见的两类生成式 AI。

## 生成式 AI 与判别式 AI

| 类型 | 目标 | 示例 |
|------|------|------|
| 判别式 AI | 判断输入属于什么类别或预测数值 | 垃圾邮件分类、房价预测 |
| 生成式 AI | 生成新的内容 | 写文章、生成图片、生成代码 |

判别式模型回答“这是什么”，生成式模型回答“可以生成什么”。

## 常见内容类型

| 内容 | 代表技术 | 应用 |
|------|----------|------|
| 文本 | GPT、Claude、Qwen、DeepSeek | 写作、问答、摘要、翻译 |
| 图像 | Stable Diffusion、DALL-E、Midjourney | 海报、插画、商品图 |
| 代码 | Codex、Copilot、代码 LLM | 代码补全、解释、重构 |
| 音频 | TTS、语音克隆、音乐生成 | 配音、播客、客服 |
| 视频 | 文生视频、图生视频 | 短视频、广告、分镜 |
| 多模态 | 视觉语言模型 | 看图问答、文档理解 |

## 生成过程的共同点

生成式模型通常不是复制已有内容，而是根据训练中学到的模式，在当前输入条件下生成最可能的输出。

```text
提示词/条件 → 模型理解 → 生成候选 → 解码/采样 → 输出内容
```

生成质量受到提示词、上下文、模型能力、采样参数和后处理影响。

## LLM 与扩散模型

| 模型类型 | 主要生成对象 | 核心直觉 |
|----------|--------------|----------|
| 大语言模型 | 文本、代码、结构化内容 | 预测下一个 token |
| 扩散模型 | 图像、视频、音频 | 从噪声逐步还原出内容 |

LLM 擅长语言、推理和工具调用；扩散模型擅长视觉生成和风格控制。

## 关键参数

| 参数 | 影响 |
|------|------|
| temperature | 越高越随机，越低越稳定 |
| top_p | 控制采样候选范围 |
| max tokens | 控制最大输出长度 |
| seed | 控制可复现性，图像生成中常见 |
| guidance scale | 图像生成中控制贴合提示词程度 |

### 采样参数的直觉理解

初学者常困惑"为什么同一提示词每次结果不同"。原因在于模型不是输出确定答案，而是输出一个概率分布，再从中"采样"：

- **temperature 低（如 0.2）**：几乎总选概率最高的 token，输出稳定、保守，适合事实问答和代码。
- **temperature 高（如 0.9）**：让低概率 token 也有机会被选中，输出多样、有创意，适合写作和头脑风暴。
- **top_p（如 0.9）**：只在累积概率前 90% 的候选词里采样，过滤掉极端小概率词。
- **图像生成特有**：`steps`（去噪步数，越多越精细越慢）、`cfg_scale`（贴合提示词程度，太高会过曝）。

> 实践建议：先固定一个参数组合跑评估集，再针对性调整。不要同时调多个参数，否则无法判断效果变化的原因。

## 风险与限制

- 幻觉：生成看似合理但错误的内容。
- 版权：训练数据和输出可能涉及版权争议。
- 隐私：输入敏感数据可能带来泄露风险。
- 偏见：模型可能继承训练数据偏见。
- 成本：长上下文、多轮调用和大图生成都会增加成本。
- 可控性：生成结果需要评估、约束和人工审核。

## 初学者实践建议

先从可验证任务开始：

- 让 LLM 摘要一篇你能核对的文章。
- 让模型把非结构化文本整理成表格。
- 用图像模型生成不同风格的同一主题。
- 比较不同提示词对输出稳定性的影响。
- 为同一个任务建立人工评分表。

## 2026 生成式 AI 最新趋势

生成式 AI 在 2026 年已经从"能否生成内容"发展到"能否交付工作"的阶段。以下八大趋势正在重塑行业格局。

### 1. Agentic AI 进入生产阶段

2025 年 Agentic AI 还停留在"演示很酷但生产不可靠"的阶段，到 2026 年中，工具调用可靠性和可观测性工具已使其在窄领域达到生产可用水平：

- **工具调用可靠性**：前沿模型的工具调用失败率大幅下降，长期任务（long-horizon）的假重试（spurious retry）显著减少
- **MCP（Model Context Protocol）**：Anthropic 推出的 MCP 协议已被广泛采用，Agent 可以跨模型复用工具
- **可观测性**：traceAI、Langfuse 等追踪工具使得 Agent 调试变得可管理

关键实践：从第一天就在 Agent 中嵌入可观测性工具，用模拟测试覆盖用户流程的超前发布，用自定义评估指标门控发布。

> 来源：Future AGI, "Generative AI Trends 2026: 8 Shifts Reshaping What Teams Build, Buy, and Replace", https://futureagi.com/blog/generative-ai-trends-2026/。

### 2. 多模态成为默认能力

2026 年，前沿模型（GPT-5、Claude Opus 4.7、Gemini 2.5 Pro）的**多模态能力已不再是"特性"而是"默认设置"**：

| 模型 | 多模态方式 | 重点能力 |
|------|-----------|----------|
| GPT-5 | 原生多模态，统一推理与通用 | 图片理解、文档分析、代码生成 |
| Claude Opus 4.7 | 图像+文本 | 长上下文编码、复杂多步 Agent 任务 |
| Gemini 2.5 Pro | 原生多模态，2M token 上下文 | 视频理解、超大文档、多模态推理 |

这意味着开发者可以**一次性调用**完成图片理解、OCR、翻译等任务，不再需要拼接多个专用模型。

### 3. 小型任务专用模型取代前沿模型做常规工作

2026 年的关键洞察：**不一定每次都用最大的模型**。Gemini Flash、GPT-5 nano、Llama 4 等小模型在常规任务上以大约一个数量级的成本优势提供了足够好的质量。

| 场景 | 推荐的模型策略 |
|------|---------------|
| 简单分类/抽取 | 小模型（GPT-5 nano、Gemini Flash） |
| 路由/初筛 | 小模型做决策，大模型做关键步骤 |
| 复杂推理/长代码 | 前沿大模型 |
| 高频低风险任务 | 小模型 + 人工抽检 |

### 4. 视频生成进入实用阶段

2026 年是 AI 视频生成爆发的一年。Sora 2、Veo 3.1、Kling 3.0、Seedance 2.0 等模型实现了接近商业级的视频质量：

| 模型 | 核心优势 | 价格参考 |
|------|---------|----------|
| Sora 2 | 物理模拟真实、长视频一致性 | $0.20/秒（1080p） |
| Veo 3.1 | 音频同步生成、多镜头 | $0.15/秒 |
| Kling 3.0 | 性价比高、中文场景表现好 | $0.05/秒 |
| Seedance 2.0 | 风格控制精细 | $0.12/秒 |

应用场景已从短视频扩展到广告、产品演示和辅助电影预可视（pre-visualization）。

> 来源：Lushbinary, "AI Video Generation 2026: Sora 2 vs Veo 3.1 vs Kling 3.0 vs Seedance 2.0", https://lushbinary.com/blog/ai-video-generation-sora-veo-kling-seedance-comparison/。

### 5. MCP 标准化工具调用

Anthropic 的 MCP（Model Context Protocol）在 2026 年被 OpenAI、Google、xAI 等主流提供商广泛采纳。这意味着：
- 开发者只需编写一次工具（MCP Server），即可在多个模型间复用
- Agent 的工具调用不再绑定特定模型供应商
- 工具发现和版本管理变得更标准化

### 6. 端侧推理成为现实

2026 年，Apple、Google Pixel、高通骁龙芯片都支持**本地运行小型生成式 AI 模型**：

- 本地设备处理简单生成任务（摘要、翻译、分类）
- 只有复杂推理才上云——降低延迟和成本
- 隐私敏感数据完全保留在设备上

### 7. 自定义评估替代公开基准

公开基准（MMLU、HumanEval 等）已经饱和，**真正有用的是你业务场景的自定义评估集**：

- 每个团队应维护 50-200 个覆盖典型用户场景的 prompt 回归测试
- 每次模型更新后自动运行评估
- 用 LLM-as-judge 评分替代人工标注（成本降低 90%+）

### 8. 多模型路由成为默认架构

2026 年的生产系统**几乎不会只绑定一个模型**：

- 主模型 + 备用模型实现故障转移
- 不同请求路由到不同模型（A/B 测试、"大模型处理复杂、小模型处理简单"）
- 安全检查（guardrails）独立于推理模型

### 给初学者的 2026 年建议

- **不要追求"最好的模型"**——没有万能模型，关键是让合适的模型做合适的事
- **多模态是标配**——即使你现在只做文本，也建议了解多模态能力边界
- **Agent 不只是噱头**——但部署时要做好可观测性、模拟测试和回滚机制
- **成本意识**——小模型比大模型便宜 10 倍以上，先评估是否真的需要大模型
- **评估驱动选型**——用你的数据做评估，不要只看排行榜
