# 大语言模型基础

> 让计算机理解并生成人类语言——大语言模型是怎么做到的？

---

## 什么是大语言模型？

**大语言模型（Large Language Model, LLM）** 是一种基于深度学习，特别是 **Transformer 架构** 训练出来的 AI 模型，它能够理解和生成人类语言。

"大"体现在三个方面：

| 维度 | 含义 | 典型数值（2025年） |
|------|------|-------------------|
| **参数量** | 模型内部可调节的权重数量 | 数十亿到数万亿 |
| **训练数据量** | 训练时使用的文本总量 | 数万亿 tokens（词元） |
| **计算资源** | 训练所需的算力 | 数千到数万 GPU 小时 |

> 💡 **类比**：LLM 就像一个读了互联网上几乎所有文字的人——它看过的材料相当于几百万本书，然后学会了如何像人类一样理解和表达语言。

---

## 大语言模型如何工作？

### 核心原理：下一个词预测

LLM 的基本训练方法出奇地简单——**预测下一个词**。

```
输入： "今天天气真"
输出： "好"（概率 85%）
       "不错"（概率 10%）
       "糟糕"（概率 3%）
       ...
```

模型通过海量文本学习，逐步掌握了单词之间的统计规律——什么词在什么语境下最可能出现。

> 💡 这就是为什么 LLM 本质上是"接龙大师"——它不断地预测下一个最合适的词，一个词一个词地生成完整的回答。

### 自注意力机制（Self-Attention）

这是 Transformer 的核心创新。它让模型在处理每个词时，能够 **关注输入序列中所有其他相关词**。

> **例子**：在句子"那只追着老鼠的猫，它很可爱"中，模型需要知道"它"指的是"猫"而不是"老鼠"。自注意力机制让模型在读到"它"时，将"猫"赋予更高的重要性。

### Transformer 架构简图

```
输入文本
    │
    ▼
┌──────────────┐
│ Tokenization │  ← 将文本拆分成词元（Token）
│  (分词)      │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  Embedding   │  ← 将词元转换为向量（数字表示）
└──────┬───────┘
       │
       ▼
┌──────────────────────────────────────────┐
│          Transformer Blocks              │
│  ┌────────────────────────────────────┐  │
│  │    Multi-Head Self-Attention       │  │  ← 理解词语之间的关系
│  └────────────────────────────────────┘  │
│                    ↓                      │
│  ┌────────────────────────────────────┐  │
│  │        Feed-Forward Network        │  │  ← 进一步处理和转换信息
│  └────────────────────────────────────┘  │
│  （重复多次 —— 几十层）                   │
└──────────────────────────────────────────┘
                      │
                      ▼
┌──────────────┐
│  Output Layer│  ← 输出下一个词的概率分布
└──────────────┘
```

## 关键概念

### Tokenization（分词）

将文本拆分成模型能处理的最小单位——**Token**（词元）。

```
"我爱学习人工智能"
→ ["我", "爱", "学习", "人工", "智能"]   ← 一个中文词可能被拆成多个 Token
```

| 概念 | 说明 |
|------|------|
| **Token** | 文本的最小处理单位（单词、子词或字符） |
| **词汇表** | 模型认识的所有 Token 的集合（通常 3万-20万个） |
| **Token 数量** | 输入/输出文本被拆成的 Token 个数（也是计费单位） |

> 💡 一个英文单词 ≈ 1 个 Token，一个中文字 ≈ 1-2 个 Token。

### 上下文窗口（Context Window）

模型一次能处理的 **最大 Token 数量**。

- **小窗口**（4K-8K）：早期模型，约 3000-6000 字
- **大窗口**（32K-128K）：当前主流模型
- **超大窗口**（1M+）：部分前沿模型（如 Gemini 1.5 Pro）

> 💡 理解为模型的"短期记忆"——窗口越大，模型能"记住"的对话历史或文档内容就越多。

### 采样参数

控制 LLM 输出方式的参数：

| 参数 | 作用 | 值越大 |
|------|------|--------|
| **Temperature**（温度） | 控制输出的随机性/创造性 | 回答越多样、越有创意 |
| **Top-p**（核采样） | 只考虑累积概率达到 p 的候选词 | 生成的候选词越多 |
| **Top-k** | 只考虑概率最高的 k 个候选词 | 选词范围越广 |

> 💡 **实际使用**：写代码时用低 Temperature（0-0.3），追求准确；写创意文案时用高 Temperature（0.7-1.0），追求多样性。

## 大语言模型的发展历程

```
GPT（2018）
  │  1.17亿参数
  ▼
GPT-2（2019）
  │  15亿参数 · 效果惊艳但最初不敢开源
  ▼
GPT-3（2020）
  │  1750亿参数 · 展示了强大的零样本/少样本能力
  ▼
ChatGPT（2022.11）
  │  基于 GPT-3.5 + RLHF · 对话界面革命
  ▼
GPT-4（2023.3）
  │  多模态 · 推理能力大幅提升
  ▼
开源模型爆发（2023-2025）
  │  LLaMA · Mistral · DeepSeek · Qwen · Yi
  │  开源模型逼近/超越闭源模型
  ▼
前沿探索（2025+）
    多模态融合 · AI Agent · 长上下文
    推理模型（o1, o3, DeepSeek-R1）
```

### 关键转折点

- **GPT-3（2020）**：证明了"规模即能力"——模型越大，涌现的能力越强
- **ChatGPT（2022）**：RLHF（基于人类反馈的强化学习）让模型更"听话"
- **开源模型（2023）**：LLaMA 系列发布，开源社区繁荣
- **推理模型（2024-2025）**：o1/R1 系列展示"慢思考"能力

## LLM 能做什么 & 不能做什么

### ✅ 擅长的事情

| 能力 | 示例 |
|------|------|
| 文本生成与创作 | 写文章、写诗、写代码 |
| 对话交流 | 回答问题、头脑风暴、聊天 |
| 代码辅助 | 代码生成、Debug、代码解释 |
| 翻译总结 | 跨语言翻译、长文本摘要 |
| 信息提取 | 从文档中提取关键信息 |
| 逻辑推理 | 数学题解答、逻辑分析（部分模型） |

### ❌ 局限与挑战

| 局限 | 说明 |
|------|------|
| **幻觉（Hallucination）** | 编造事实，言之凿凿地给出错误信息 |
| **知识截止** | 只学习到训练数据截止日期之前的知识 |
| **缺乏真正理解** | 没有意识和主观体验 |
| **数学不精确** | 复杂算术和推理仍会出错 |
| **上下文有限** | 超出窗口的内容会被"遗忘" |
| **偏见与安全** | 可能继承训练数据中的偏见 |

## 理解提示（Prompt）

**提示（Prompt）** 是你给 LLM 的指令或输入。好的 Prompt 能让模型给出更好的回答。

### Prompt 编写技巧

```
❌ 不好的 Prompt:
"写个故事"

✅ 好的 Prompt:
"写一个 200 字的科幻短故事，主题是'人类与 AI 机器人成为朋友'，
风格温暖治愈，适合儿童阅读。"
```

### Prompt 结构建议

- **明确角色**：`你是一个专业的 Python 程序员`
- **给出上下文**：`我有以下代码，请帮我解释`
- **指定格式**：`请用表格形式回答`
- **设置限制**：`请控制在 300 字以内`

## 伦理考虑

随着 LLM 能力的增强，我们需要关注以下问题：

| 问题 | 说明 |
|------|------|
| **信息真实性** | LLM 可能传播错误信息，需要人工验证 |
| **偏见与歧视** | 训练数据中的偏见可能被放大 |
| **隐私安全** | 用户输入的敏感信息可能被记录 |
| **滥用风险** | 深度伪造、虚假信息、学术作弊 |
| **就业影响** | 部分工作可能被替代，但也创造新机会 |
| **环境影响** | 训练大模型消耗大量电力资源 |

> 💡 **正确态度**：LLM 是强大的工具，但需要负责任地使用——验证信息、保护隐私、遵守伦理规范。

## ✅ 本章小结

- **大语言模型（LLM）** 通过预测下一个词来学习理解和生成语言
- 核心是 **Transformer 架构** 和 **自注意力机制**
- 关键概念：Tokenization、上下文窗口、Temperature、Top-p
- LLM 从 GPT 系列发展到今天的百花齐放
- LLM 擅长生成、翻译、编程等任务，但也有幻觉、偏见等局限
- 好的 **Prompt** 能显著提升 LLM 的输出质量
- 使用大模型需要注意伦理和安全问题
