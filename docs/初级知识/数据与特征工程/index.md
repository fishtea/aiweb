# 数据与特征工程

数据决定了 AI 系统的上限。很多项目失败不是因为模型不够先进，而是因为数据质量差、标签不清晰、特征设计不合理或训练数据与真实场景不一致。

## 数据在 AI 项目中的位置

```text
业务问题 → 数据定义 → 数据采集 → 数据清洗 → 特征工程 → 模型训练 → 评估 → 上线监控
```

模型只能从数据中学习已经存在的模式。如果数据本身有偏差、噪声或缺口，模型会把这些问题一起学进去。

## 数据类型

| 数据类型 | 示例 | 常见任务 |
|----------|------|----------|
| 表格数据 | 用户、订单、交易、传感器记录 | 分类、回归、异常检测 |
| 文本数据 | 评论、文章、客服对话、合同 | 分类、摘要、问答、检索 |
| 图像数据 | 商品图、医学影像、监控画面 | 分类、检测、分割 |
| 音频数据 | 语音、音乐、环境声音 | 语音识别、说话人识别 |
| 行为序列 | 点击流、购买历史、浏览路径 | 推荐、预测、风控 |

## 标签是什么

监督学习需要标签。标签是模型要学习预测的目标。

| 任务 | 输入 | 标签 |
|------|------|------|
| 垃圾邮件识别 | 邮件内容 | 垃圾/正常 |
| 房价预测 | 面积、位置、楼龄 | 成交价格 |
| 评论情感分析 | 评论文本 | 正面/中性/负面 |
| 医学影像分类 | 影像图片 | 是否异常 |

标签质量很关键。含糊、冲突或不一致的标签会让模型学习到错误规则。

## 特征工程

特征是模型用来做判断的输入信息。特征工程包括选择、清洗、转换和创造特征。

| 操作 | 说明 | 示例 |
|------|------|------|
| 缺失值处理 | 补齐或删除缺失数据 | 年龄缺失填中位数 |
| 类别编码 | 把类别转成数字 | 城市、职业、设备类型 |
| 标准化 | 调整数值尺度 | 收入、面积、访问次数 |
| 文本向量化 | 把文本转成向量 | TF-IDF、Embedding |
| 时间特征 | 提取时间规律 | 星期几、是否节假日 |
| 聚合特征 | 从历史行为中总结信息 | 过去 7 天购买次数 |

## 数据泄漏

数据泄漏是初学者最容易忽视的问题：训练数据中包含了真实预测时拿不到的信息。

例子：

- 用“订单是否退款”预测“用户是否会退款”，但这个字段本身在预测时还不存在。
- 先对全量数据做标准化，再划分训练集和测试集。
- 同一个用户的高度相似记录同时出现在训练集和测试集。

数据泄漏会让测试分数虚高，上线后效果突然变差。

## 数据划分

常见划分方式：

| 数据集 | 用途 |
--------|------|
| 训练集 | 用来训练模型参数 |
| 验证集 | 用来调参、选择模型 |
| 测试集 | 用来模拟最终泛化能力 |

时间序列任务通常不能随机划分，应按时间切分，避免未来数据进入训练过程。

## 数据质量检查清单

- 字段含义是否明确？
- 标签是否一致、可靠、可复核？
- 缺失值和异常值比例是否可接受？
- 训练数据是否覆盖真实场景？
- 正负样本是否严重不均衡？
- 是否存在重复样本或数据泄漏？
- 敏感信息是否已经脱敏？

## 数值特征归一化

不同特征往往量纲差异很大（如年龄 0-100 与收入 0-1000000）。如果不做归一化，模型会过度关注数值范围大的特征。归一化将不同尺度的数值转换到相似区间。

### 三种主流归一化方法

| 方法 | 公式 | 适用场景 | 注意事项 |
|------|------|----------|----------|
| 线性缩放（Min-Max Scaling） | \(x' = (x - x_{min}) / (x_{max} - x_{min})\) | 数据范围已知且无明显异常值 | 对新数据中的越界值敏感 |
| Z-Score 标准化 | \(x' = (x - \mu) / \sigma\) | 数据近似正态分布 | 不保证输出在固定区间内 |
| 对数缩放（Log Scaling） | \(x' = \log(x + 1)\) | 数据右偏严重（如收入、访问量） | 零值处理需加 1 |

### 何时必须归一化

- 使用梯度下降优化的模型（线性回归、神经网络、SVM）：**必须归一化**，否则梯度下降会震荡，收敛速度很慢。
- 基于树的方法（决策树、随机森林、GBDT）：不受特征尺度影响，归一化不是必须的。
- KNN 和 K-Means：**推荐归一化**，因为这些算法依赖距离计算。

### 异常值处理（Clipping）

当数值特征中存在极端值时，归一化前先做截断（Clipping）：

```
# 将超过上限的值截断为上限值
x = min(max(x, lower_bound), upper_bound)
```

常用方法：基于百分位数（如取 1%-99% 范围）或基于标准差（如 ±3σ 范围）。先截断再归一化，可以防止个别异常值主导整个特征。

## 类别特征编码

机器学习模型只能处理数值。类别特征（如城市、颜色、职业）需要转换为数值表示。

### 词汇表编码（Vocabulary Encoding）

将每个类别映射到一个唯一整数：

```
颜色 → {红色: 0, 蓝色: 1, 绿色: 2, 白色: 3, ...}
```

⚠️ **注意**：如果直接将整数索引喂给模型，模型会认为"绿色(2) = 红色(0) × 2"，产生错误的数值关系。因此词汇表编码后通常需要配合 Embedding 层或独热编码。

### 独热编码（One-Hot Encoding）

这是最常用的类别编码方法。每个类别被表示为一个向量，长度等于类别总数，只有对应位置为 1，其余为 0。

| 颜色 | 编码向量 |
|------|----------|
| 红色 | [1, 0, 0, 0, 0, ...] |
| 蓝色 | [0, 1, 0, 0, 0, ...] |
| 绿色 | [0, 0, 1, 0, 0, ...] |

**适用场景**：类别数较少（如几十个以内）时效果很好。如果类别数成百上千（如邮政编码），独热编码会产生过大的稀疏向量。此时建议使用 Embedding 或哈希编码。

**稀疏表示**：独热编码向量中大部分元素是 0，可以通过稀疏向量格式存储（只记录非零元素的位置），大幅减少内存占用。

### 类别数据中的异常值

类别特征也可能包含异常值——那些出现次数极少的稀有类别。处理方式：

- 阈值合并：出现次数小于 N 次的类别统一归为"其他"
- 哈希编码：将类别映射到固定长度的哈希桶中
- Embedding 层：在神经网络中用嵌入向量学习类别的语义关系

## 特征选择

特征选择是决定哪些特征应该参与训练的过程。特征过多会导致过拟合和训练变慢；特征过少会丢失重要信息。

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 过滤法 | 用统计指标（相关系数、卡方检验、互信息）筛选特征 | 快速初筛，独立于模型 |
| 包裹法 | 用模型表现评估特征子集（如递归特征消除） | 小特征集时效果好，计算量大 |
| 嵌入法 | 训练时自动做特征选择（如 L1 正则化、树模型的特征重要性） | 最常用，效果与计算量的平衡好 |

实践中建议先用嵌入法（L1 正则化或树模型）快速筛选出重要特征，再用领域知识做最终决定。

## 2026 年数据工程与特征工程新趋势

### 概述

数据工程在 2026 年已从"数据清洗"演变为涵盖**数据治理、合成数据、数据飞轮和评估驱动开发**的完整体系。AI 项目的瓶颈不再是模型能力，而是数据质量、标注成本和特征管理。

### 核心要点

#### 1. 合成数据成为主流数据源

2025-2026 年，生成式 AI 产出的合成数据在质量上已达到与真实数据难以区分的水平。对于目标检测、缺陷识别、NLP 标注等场景，合成数据可以将数据准备时间从数月缩短到数天，成本降低约 40%。

**关键场景**：
- **长尾覆盖**：真实世界罕见的边缘案例（如罕见交通事故、工业异常）可以通过程序化生成
- **隐私保护**：用合成数据替代敏感真实数据（如医疗影像、财务记录）
- **数据增强**：扩散模型（Stable Diffusion、DALL-E）可生成指定场景的训练图像

> 参考：discoverinai.com, "Computer Vision: 5 Key 2026 Trends", https://discoverinai.com/computer-vision-5-key-2026-trends-to-watch/

#### 2. 数据飞轮（Data Flywheel）成为标准实践

单一的数据采集 → 清洗 → 训练 → 部署管线已被淘汰。2026 年的标准实践是**数据飞轮**——模型在生产中的表现数据反馈回数据工程管线，持续改进训练数据质量：

```
部署模型 → 收集推理日志 → 识别失败案例 → 人工标注 → 补充训练数据 → 重新训练 → 部署新版本
```

每个循环都让模型在真实分布上表现更好。关键基础设施包括：
- **推理日志系统**：记录每次推理的输入、输出、置信度和错误标记
- **失败案例库**：自动收集置信度低、用户手动修正、下游检测失败的样本
- **标注反馈回路**：将失败案例优先分配给人工标注或合成数据生成

#### 3. 评估驱动开发（Evaluation-Driven Development）

2026 年的一个关键转变是：**在开始标注数据之前，先设计评估集**。公开基准（MMLU、HumanEval 等）已饱和，真正有用的是业务场景的自定义评估集：

- 每个团队维护 50-200 个覆盖典型用户场景的 prompt 回归测试
- 每次模型更新后自动运行评估
- 失败案例自动进入数据飞轮，形成持续改进闭环

> 参考：Future AGI, "Generative AI Trends 2026: 8 Shifts", https://futureagi.com/blog/generative-ai-trends-2026/

#### 4. 特征存储（Feature Store）管理特征生命周期

当特征数量超过几十个时，手动管理特征定义、版本和一致性问题变得困难。**特征存储**（如 Feast、Tecton）解决了三个关键问题：

| 问题 | 后果 | 特征存储方案 |
|------|------|-------------|
| 训练与推理时特征不一致 | 线上效果远低于离线评估 | 统一特征定义和计算逻辑 |
| 特征没有版本控制 | 模型回滚时特征不兼容 | 特征按时间戳版本化 |
| 特征重复开发 | 不同团队重复造相同特征 | 特征注册中心，一次定义全局复用 |

#### 5. 数据标注的范式转变

2026 年标注方式正在从"全人工标注"向"AI 辅助标注 + 人工校验"转变：

| 标注方式 | 适用场景 | 成本对比 |
|----------|----------|:--------:|
| 全人工标注（传统） | 高质量标准数据集 | 1x（基准） |
| AI 预标注 + 人工修正 | 通用视觉/NLP 任务 | 0.3-0.5x |
| 主动学习（Active Learning） | 数据量大但标注预算有限 | 0.2-0.4x |
| 合成数据 + 自动标注 | 大规模训练数据快速生产 | 0.05-0.2x |
| 弱监督（Weak Supervision） | 有规则的场景（如医疗 ICD 编码） | 0.1-0.3x |

#### 6. 数据质量自动化检查

2026 年的数据管线不再依赖人工审查质量，而是**自动化的数据质量检查**：

- **缺失率监控**：特征缺失率超过阈值时自动告警
- **分布漂移检测**：线上数据分布与训练集分布差异过大时触发重训练
- **数据泄漏检测**：自动扫描训练数据中是否混入未来信息
- **标签质量审计**：抽检标注一致性，计算标注者间信度（Inter-annotator Agreement）
- **数据血缘追踪**：每个样本的来源、转换历史全程可追溯

### 初学者实践建议

1. **从干净的现有数据集开始**：Kaggle 和 HuggingFace Datasets 有大量高质量公开数据集，先学会用好数据，再学处理脏数据
2. **养成数据探索习惯**：训练任何模型前，先用 `df.describe()`、`df.info()`、`df.isnull().sum()` 和可视化检查数据
3. **建立简单的基线**：先用最简单的方法（如均值填充、独热编码、逻辑回归）作为基线，再逐步优化
4. **先设计评估集，再标注训练数据**：明确成功的衡量标准后再投入标注成本
5. **记录每个数据转换步骤**：确保训练和推理使用完全相同的数据处理逻辑
