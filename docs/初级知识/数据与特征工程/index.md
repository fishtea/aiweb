# 数据与特征工程

数据决定了 AI 系统的上限。很多项目失败不是因为模型不够先进，而是因为数据质量差、标签不清晰、特征设计不合理或训练数据与真实场景不一致。

## 数据在 AI 项目中的位置

```text
业务问题 → 数据定义 → 数据采集 → 数据清洗 → 特征工程 → 模型训练 → 评估 → 上线监控
```

模型只能从数据中学习已经存在的模式。如果数据本身有偏差、噪声或缺口，模型会把这些问题一起学进去。

## 数据类型

| 数据类型 | 示例 | 常见任务 |
|----------|------|----------|
| 表格数据 | 用户、订单、交易、传感器记录 | 分类、回归、异常检测 |
| 文本数据 | 评论、文章、客服对话、合同 | 分类、摘要、问答、检索 |
| 图像数据 | 商品图、医学影像、监控画面 | 分类、检测、分割 |
| 音频数据 | 语音、音乐、环境声音 | 语音识别、说话人识别 |
| 行为序列 | 点击流、购买历史、浏览路径 | 推荐、预测、风控 |

## 标签是什么

监督学习需要标签。标签是模型要学习预测的目标。

| 任务 | 输入 | 标签 |
|------|------|------|
| 垃圾邮件识别 | 邮件内容 | 垃圾/正常 |
| 房价预测 | 面积、位置、楼龄 | 成交价格 |
| 评论情感分析 | 评论文本 | 正面/中性/负面 |
| 医学影像分类 | 影像图片 | 是否异常 |

标签质量很关键。含糊、冲突或不一致的标签会让模型学习到错误规则。

## 特征工程

特征是模型用来做判断的输入信息。特征工程包括选择、清洗、转换和创造特征。

| 操作 | 说明 | 示例 |
|------|------|------|
| 缺失值处理 | 补齐或删除缺失数据 | 年龄缺失填中位数 |
| 类别编码 | 把类别转成数字 | 城市、职业、设备类型 |
| 标准化 | 调整数值尺度 | 收入、面积、访问次数 |
| 文本向量化 | 把文本转成向量 | TF-IDF、Embedding |
| 时间特征 | 提取时间规律 | 星期几、是否节假日 |
| 聚合特征 | 从历史行为中总结信息 | 过去 7 天购买次数 |

## 数据泄漏

数据泄漏是初学者最容易忽视的问题：训练数据中包含了真实预测时拿不到的信息。

例子：

- 用“订单是否退款”预测“用户是否会退款”，但这个字段本身在预测时还不存在。
- 先对全量数据做标准化，再划分训练集和测试集。
- 同一个用户的高度相似记录同时出现在训练集和测试集。

数据泄漏会让测试分数虚高，上线后效果突然变差。

## 数据划分

常见划分方式：

| 数据集 | 用途 |
--------|------|
| 训练集 | 用来训练模型参数 |
| 验证集 | 用来调参、选择模型 |
| 测试集 | 用来模拟最终泛化能力 |

时间序列任务通常不能随机划分，应按时间切分，避免未来数据进入训练过程。

## 数据质量检查清单

- 字段含义是否明确？
- 标签是否一致、可靠、可复核？
- 缺失值和异常值比例是否可接受？
- 训练数据是否覆盖真实场景？
- 正负样本是否严重不均衡？
- 是否存在重复样本或数据泄漏？
- 敏感信息是否已经脱敏？

## 数值特征归一化

不同特征往往量纲差异很大（如年龄 0-100 与收入 0-1000000）。如果不做归一化，模型会过度关注数值范围大的特征。归一化将不同尺度的数值转换到相似区间。

### 三种主流归一化方法

| 方法 | 公式 | 适用场景 | 注意事项 |
|------|------|----------|----------|
| 线性缩放（Min-Max Scaling） | \(x' = (x - x_{min}) / (x_{max} - x_{min})\) | 数据范围已知且无明显异常值 | 对新数据中的越界值敏感 |
| Z-Score 标准化 | \(x' = (x - \mu) / \sigma\) | 数据近似正态分布 | 不保证输出在固定区间内 |
| 对数缩放（Log Scaling） | \(x' = \log(x + 1)\) | 数据右偏严重（如收入、访问量） | 零值处理需加 1 |

### 何时必须归一化

- 使用梯度下降优化的模型（线性回归、神经网络、SVM）：**必须归一化**，否则梯度下降会震荡，收敛速度很慢。
- 基于树的方法（决策树、随机森林、GBDT）：不受特征尺度影响，归一化不是必须的。
- KNN 和 K-Means：**推荐归一化**，因为这些算法依赖距离计算。

### 异常值处理（Clipping）

当数值特征中存在极端值时，归一化前先做截断（Clipping）：

```
# 将超过上限的值截断为上限值
x = min(max(x, lower_bound), upper_bound)
```

常用方法：基于百分位数（如取 1%-99% 范围）或基于标准差（如 ±3σ 范围）。先截断再归一化，可以防止个别异常值主导整个特征。

## 类别特征编码

机器学习模型只能处理数值。类别特征（如城市、颜色、职业）需要转换为数值表示。

### 词汇表编码（Vocabulary Encoding）

将每个类别映射到一个唯一整数：

```
颜色 → {红色: 0, 蓝色: 1, 绿色: 2, 白色: 3, ...}
```

⚠️ **注意**：如果直接将整数索引喂给模型，模型会认为"绿色(2) = 红色(0) × 2"，产生错误的数值关系。因此词汇表编码后通常需要配合 Embedding 层或独热编码。

### 独热编码（One-Hot Encoding）

这是最常用的类别编码方法。每个类别被表示为一个向量，长度等于类别总数，只有对应位置为 1，其余为 0。

| 颜色 | 编码向量 |
|------|----------|
| 红色 | [1, 0, 0, 0, 0, ...] |
| 蓝色 | [0, 1, 0, 0, 0, ...] |
| 绿色 | [0, 0, 1, 0, 0, ...] |

**适用场景**：类别数较少（如几十个以内）时效果很好。如果类别数成百上千（如邮政编码），独热编码会产生过大的稀疏向量。此时建议使用 Embedding 或哈希编码。

**稀疏表示**：独热编码向量中大部分元素是 0，可以通过稀疏向量格式存储（只记录非零元素的位置），大幅减少内存占用。

### 类别数据中的异常值

类别特征也可能包含异常值——那些出现次数极少的稀有类别。处理方式：

- 阈值合并：出现次数小于 N 次的类别统一归为"其他"
- 哈希编码：将类别映射到固定长度的哈希桶中
- Embedding 层：在神经网络中用嵌入向量学习类别的语义关系

## 特征选择

特征选择是决定哪些特征应该参与训练的过程。特征过多会导致过拟合和训练变慢；特征过少会丢失重要信息。

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 过滤法 | 用统计指标（相关系数、卡方检验、互信息）筛选特征 | 快速初筛，独立于模型 |
| 包裹法 | 用模型表现评估特征子集（如递归特征消除） | 小特征集时效果好，计算量大 |
| 嵌入法 | 训练时自动做特征选择（如 L1 正则化、树模型的特征重要性） | 最常用，效果与计算量的平衡好 |

实践中建议先用嵌入法（L1 正则化或树模型）快速筛选出重要特征，再用领域知识做最终决定。
