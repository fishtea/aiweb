# 数学基础

学习 AI 不需要一开始就掌握高等数学全貌，但需要理解几个核心数学工具。它们帮助你看懂模型如何表示数据、如何学习、如何判断好坏。

## 最小必要数学

| 模块 | 为什么重要 | 入门重点 |
|------|------------|----------|
| 线性代数 | AI 把文本、图片、用户行为表示成向量和矩阵 | 向量、矩阵乘法、维度、点积 |
| 概率统计 | 模型输出通常是概率，评估也依赖统计 | 均值、方差、概率分布、条件概率 |
| 微积分 | 神经网络通过梯度下降优化参数 | 导数、梯度、链式法则 |
| 优化方法 | 训练模型就是最小化损失函数 | 损失函数、学习率、局部最优 |
| 信息论 | 分类、语言模型和压缩都涉及不确定性 | 熵、交叉熵、困惑度 |

## 向量和矩阵

AI 系统通常不会直接处理“文字”或“图片”，而是先把它们变成数字。

```text
一条用户评论 → 文本向量
一张图片 → 像素矩阵或视觉特征向量
一个用户 → 年龄、城市、行为次数等特征向量
```

向量可以理解为一组有顺序的数字。两个向量越相似，通常表示它们代表的对象越接近。Embedding、推荐系统、向量数据库都建立在这个思想上。

## 概率思维

模型很少给出绝对确定的答案。分类模型常输出每个类别的概率：

| 类别 | 概率 |
|------|------|
| 正面 | 0.82 |
| 中性 | 0.13 |
| 负面 | 0.05 |

概率最高的类别不一定永远正确，它只代表模型在当前输入下的置信程度。实际系统还需要阈值、人工复核和错误处理。

## 损失函数

损失函数用来衡量模型预测和真实答案之间的差距。训练过程的目标是让损失越来越小。

| 任务 | 常见损失 | 直觉 |
|------|----------|------|
| 回归 | 均方误差 MSE | 预测值离真实值越远，惩罚越大 |
| 分类 | 交叉熵 | 正确类别概率越低，惩罚越大 |
| 语言模型 | 下一个 token 预测损失 | 预测真实下一个 token 的能力 |

## 梯度下降

梯度下降可以理解为"沿着损失下降最快的方向调整参数"。

```text
初始化参数 → 计算预测 → 计算损失 → 计算梯度 → 更新参数 → 重复
```

学习率决定每次更新走多远。学习率太大，训练可能震荡；学习率太小，训练会非常慢。

## 2025-2026 AI 数学新视角

### 深度学习让数学直觉比公式更重要

2025-2026 年，随着 AutoML、LoRA、QLoRA 等工具越来越成熟，入门 AI 对纯数学推导的要求比以往更低，但**对数学直觉和维度理解的要求反而更高**。

**核心变化**：

| 过去 | 现在 |
|------|------|
| 从零推导反向传播公式 | 理解梯度回传的方向和含义 |
| 手写矩阵求导 | 理解矩阵维度变换（shape tracking） |
| 自己实现优化器 | 理解何时用 AdamW、何时用 SGD |
| 数学证明模型收敛性 | 通过训练曲线判断是否收敛 |

### 线性代数的核心应用

**向量维度**是 2026 年 AI 工程师最常遇到的数学概念：

- **Embedding 维度**：text-embedding-3-small（1536 维）、BGE（1024 维）——维度决定检索精度和存储成本
- **注意力机制**：Q/K/V 矩阵乘法的维度对齐是 Transformer 最核心的数学操作
- **LoRA 低秩适配**：用低秩矩阵分解（r=8~64）近似全量微调——理解秩（Rank）的概念直接关系微调效果
- **多头注意力**：将高维拆成多个子空间并行计算，再拼接回去

**实用技巧**：不要怕矩阵公式，先从维度标注开始——`[batch, seq_len, d_model]` × `[d_model, d_out]` = `[batch, seq_len, d_out]`，养成看维度是否匹配的习惯。

### 概率与不确定性

AI 系统本质上都在处理不确定性：

- **软概率输出**：分类模型输出 softmax 概率分布，0.7 不代表"70% 正确"，只代表相对置信度
- **贝叶斯思维**：先验（pre-trained 知识）+ 证据（new data）= 后验（微调后模型）
- **困惑度（Perplexity）**：语言模型的核心指标，等价于模型对下一个 token 的不确定性度量
- **温度参数（Temperature）**：控制输出的随机性——温度低趋近贪婪，温度高增加多样性

### 损失函数深度理解

除了基础的交叉熵和 MSE，2026 年常用损失函数：

| 损失函数 | 应用场景 | 数学直觉 |
|----------|----------|----------|
| **交叉熵** | 分类、语言模型 | 预测分布与真实分布的差异 |
| **对比损失（Contrastive Loss）** | CLIP、Embedding 训练 | 正样本对距离近，负样本对距离远 |
| **Triplet Loss** | 人脸识别、检索 | anchor-positive 距离 < anchor-negative 距离 |
| **KL 散度** | 蒸馏、RLHF | 两个概率分布的差异度量 |
| **DPO 损失** | 偏好对齐 | 直接优化偏好，不需要单独奖励模型 |

### 现代优化方法

- **AdamW**：目前最常用的优化器，比 SGD 更快收敛，带权重衰减防止过拟合
- **Cosine Annealing**：学习率按余弦曲线衰减，比阶梯衰减更平滑
- **Warmup**：训练初期用小学习率稳定梯度，再升到目标值
- **Gradient Checkpointing**：用时间换空间，减少显存占用
- **混合精度训练**：FP16/BF16 计算 + FP32 参数存储，速度翻倍

### 关键认知转变

2025-2026 年，随着 LoRA、QLoRA、AutoML 等工具的成熟，入门 AI 对公式推导的要求降低，但对**数学直觉和维度理解**的要求反而提高了。

### 线性代数：AI 工程师最频繁使用的数学

**维度追踪**是 2026 年 AI 工程师最常遇到的数学概念：

- **Embedding 维度**：text-embedding-3-small（1536 维）、BGE-M3（1024 维）——维度直接影响检索精度和存储成本
- **注意力机制**：Q/K/V 矩阵乘法的维度对齐是 Transformer 的核心操作，`[batch, seq_len, d_model] × [d_model, d_out]` 的维度匹配是调试 Transformer 代码的日常任务
- **LoRA 低秩适配**：用低秩矩阵分解（`r=8~64`）近似全量微调——理解**秩（Rank）** 的意义直接关系微调效果
- **多头注意力**：将高维拆成多个子空间并行计算，再拼接回去

> **实用技巧**：不要怕矩阵公式，先从「维度的变化」开始理解——每次矩阵乘法都会改变形状，养成标注维度的习惯。

### 概率论：模型输出的本质

LLM 的核心是概率预测，不是确定性推理：

- **Softmax 输出**：模型输出的是概率分布，0.7 不代表"70% 正确"，只代表"相对置信度"
- **贝叶斯思维**：先验（预训练知识）+ 证据（新数据）= 后验（微调后模型）
- **困惑度（Perplexity）**：语言模型的核心指标，等价于模型对下一个 token 的不确定性
- **Temperature**：控制概率分布的"锐度"——温度低 → 趋近贪婪采样，温度高 → 增加多样性
- **对比学习损失**：CLIP、Sentence-BERT 等模型用对比损失拉近相似样本的距离、推远不相似样本

### 微积分：梯度下降的实际理解

不需要精通链式法则推导，但需要理解：

- **梯度**：微分的向量形式，指向损失函数增长最快的方向。梯度下降沿反方向更新参数
- **链式法则**：反向传播的基础——误差从输出层逐层传回输入层
- **梯度消失/爆炸**：深层网络中梯度过小（停止学习）或过大（训练震荡），LayerNorm 和残差连接是解决方案
- **学习率调度**：Cosine Annealing 是 2026 年最常用的调度策略，Warmup + 余弦衰减的组合是标准配置

### 实际应用：从代码理解数学

以下 PyTorch 代码展示了数学概念如何在实际中应用：

```python
import torch
import torch.nn.functional as F

# 线性代数：矩阵乘法即全连接层
x = torch.randn(32, 768)           # batch=32, 特征维度=768
W = torch.randn(768, 2048)         # 权重矩阵
h = x @ W                          # [32, 768] @ [768, 2048] = [32, 2048]

# 概率：Softmax 将 logits 转为概率分布
logits = torch.randn(32, 10)       # 10 个类别的原始分数
probs = F.softmax(logits, dim=-1)  # 每行和为 1，表示概率分布

# 微积分：自动求导计算梯度
x = torch.tensor([3.0], requires_grad=True)
y = x**2 + 2*x + 1
y.backward()                       # dy/dx = 2x + 2 = 8
print(x.grad)                      # tensor([8.])

# 损失函数：交叉熵
loss = F.cross_entropy(logits, torch.randint(0, 10, (32,)))
```

## 初学者常见误区

- 不必先把所有公式推导完再学应用，先理解直觉和用途。
- 维度比公式更重要，很多代码错误来自矩阵维度不匹配。
- 概率不是保证，模型输出 90% 仍可能出错。
- 训练损失下降不代表模型在真实场景中一定更好。

## 学习建议

先用图形和代码理解概念，再补公式。推荐顺序是：

1. 向量、矩阵、点积、余弦相似度
2. 均值、方差、正态分布、条件概率
3. 导数、偏导数、梯度
4. 损失函数、梯度下降、学习率
5. 交叉熵、softmax、困惑度
