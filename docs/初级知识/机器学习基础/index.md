# 机器学习基础

> 机器是如何"学习"的？理解机器学习的基本原理和方法

---

## 什么是机器学习？

**机器学习（Machine Learning, ML）** 是人工智能的核心分支，它让计算机 **从数据中自动学习规律和模式**，而不需要被显式地编程每一步该怎么做。

### 传统编程 vs 机器学习

```
传统编程：
  数据 + 规则  ──→  输出结果

机器学习：
  数据 + 结果  ──→  学习出规则（模型）
```

> 💡 **类比**：传统编程就像给一个人一本菜谱，告诉他每一步怎么做。机器学习就像给一个人看 100 道菜的照片和做法，让他自己总结出做菜的规律。

### 一个直观的例子：垃圾邮件过滤

- **传统方式**：程序员写规则——"如果邮件包含'中奖'、'免费'这些词，就标记为垃圾邮件"
- **机器学习方式**：给模型展示 10000 封已标记好的邮件（垃圾/正常），模型自己学会识别垃圾邮件的模式

机器学习的优势在于：它能发现人类难以显式描述的复杂模式，并且随着数据增多，效果会越来越好。

## 机器学习的三大类型

### 1️⃣ 监督学习（Supervised Learning）

**有标签**的数据学习。就像学生做练习题，每道题都有标准答案。

- **工作原理**：给定输入 → 输出对（特征 → 标签），模型学习它们之间的映射关系
- **常见任务**：
  - **分类（Classification）**：预测类别——这张图片是猫还是狗？这封邮件是垃圾还是正常？
  - **回归（Regression）**：预测数值——明天的房价是多少？下个月的销售额是多少？
- **经典算法**：线性回归、逻辑回归、决策树、支持向量机（SVM）、K 近邻（kNN）

```
监督学习示例：
  输入（特征）          →  输出（标签）
  [面积:120㎡, 卧室:3]  →  价格: 450万
  [面积:85㎡,  卧室:2]  →  价格: 280万
  [面积:150㎡, 卧室:4]  →  价格: 600万
```

### 2️⃣ 无监督学习（Unsupervised Learning）

**没有标签**的数据学习。模型自己发现数据中的结构或模式。

- **工作原理**：只给输入数据，不给答案，模型自己找出规律
- **常见任务**：
  - **聚类（Clustering）**：将相似的数据分到一组——客户分群、新闻分类
  - **降维（Dimensionality Reduction）**：简化数据的同时保留关键信息——数据可视化
- **经典算法**：K-Means 聚类、主成分分析（PCA）

```
无监督学习示例：
  给出一堆水果的图片（无标签）
  → 模型自动分成"红色的圆的水果"和"黄色的长条水果"
  （虽然它不知道这叫"苹果"和"香蕉"，但它发现了类别）
```

### 3️⃣ 强化学习（Reinforcement Learning）

通过 **与环境交互并获得奖励** 来学习。就像训练小狗——做对了给零食，做错了没有。

- **工作原理**：智能体（Agent）在环境中行动，根据行动结果获得奖励或惩罚，目标是最大化总奖励
- **典型应用**：AlphaGo、自动驾驶、游戏 AI
- **经典算法**：Q-Learning、Deep Q-Network（DQN）、PPO

> 💡 初级阶段的重点是 **监督学习和无监督学习**，强化学习在高级阶段深入学习。

## 核心概念

### 特征（Features）与标签（Labels）

| 概念 | 定义 | 房价预测例子 |
|------|------|-------------|
| **特征** | 用来预测的输入变量 | 面积、卧室数量、位置 |
| **标签** | 要预测的目标变量 | 最终成交价格 |
| **样本** | 一个完整的数据点 | 某一套房子的全部信息 |

### 训练集（Training Set）与测试集（Test Set）

将数据分为两部分：

- **训练集**（通常 80%）：用来训练模型，让模型学习规律
- **测试集**（通常 20%）：用来评估模型在"新数据"上的表现

> ⚠️ 模型在测试集上的表现才是真正的能力——就像考试要用没做过的题来测试，不能考原题。

### 过拟合（Overfitting）

模型在训练数据上表现极好，但在新数据上表现很差。

> 💡 **类比**：一个学生把练习册的答案都背下来了，但考到没见过的题目就不会做——这就是"过拟合"。
>
> **解决方式**：使用更多训练数据、简化模型、使用正则化技术

### 欠拟合（Underfitting）

模型连训练数据都没有学好，表现不佳。

> 💡 **类比**：学生根本没认真学，连简单的题都不会做。

## 常见算法速览

| 算法 | 类型 | 用途 | 简单理解 |
|------|------|------|----------|
| **线性回归** | 监督（回归） | 预测连续值 | 找一条最能拟合数据点的直线 |
| **逻辑回归** | 监督（分类） | 二分类问题 | 计算属于某类的概率 |
| **决策树** | 监督 | 分类/回归 | 像"如果-那么"的流程图 |
| **随机森林** | 监督 | 分类/回归 | 多棵决策树投票，更稳定 |
| **SVM** | 监督 | 分类 | 找到分隔不同类别的最佳边界 |
| **K 近邻（kNN）** | 监督 | 分类/回归 | 看离它最近的 K 个邻居是什么 |
| **K-Means** | 无监督 | 聚类 | 把数据分成 K 个组 |

> 💡 **初学者建议**：先理解"每个算法用来解决什么问题"以及"它的基本思想是什么"，不必深究数学推导。

## 评估指标

### 分类任务指标

| 指标 | 含义 | 适用场景 |
|------|------|----------|
| **准确率（Accuracy）** | 预测正确的比例 | 各类别数量均衡时 |
| **精确率（Precision）** | 预测为"正"的里面有多少是真的"正" | 垃圾邮件检测（避免误删正常邮件） |
| **召回率（Recall）** | 真正的"正"里有多少被找到了 | 癌症检测（尽量不漏诊） |
| **F1 分数** | 精确率和召回率的调和平均 | 需要同时兼顾精确和召回 |

### 回归任务指标

| 指标 | 含义 |
|------|------|
| **MSE（均方误差）** | 预测值与真实值差值的平方的平均值 |
| **MAE（平均绝对误差）** | 预测值与真实值差值的绝对值的平均值 |
| **R²（决定系数）** | 模型能解释数据中多少比例的变异 |

## 机器学习的标准工作流程

一个完整的机器学习项目通常包含以下步骤：

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 1. 数据  │    │ 2. 数据  │    │ 3. 选择  │    │ 4. 训练  │    │ 5. 评估  │
│  收集   ──→   │  预处理  ──→   │   模型   ──→   │   模型   ──→   │   模型   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                                    │
                                                                    ▼
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 8. 监控  │    │ 7. 部署  │    │ 6. 调优  │    │ (不满意)  │
│  与维护  │←──│   上线   │←──│  超参数  │←──│  重新优化 │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 各步骤说明

1. **数据收集** — 收集足够多的高质量数据
2. **数据预处理** — 清洗数据、处理缺失值、特征工程
3. **选择模型** — 根据问题类型选择合适的算法
4. **训练模型** — 用训练数据让模型学习
5. **评估模型** — 在测试集上评估模型效果
6. **调优** — 调整模型参数提升效果
7. **部署上线** — 将模型应用到实际场景
8. **监控维护** — 持续监控模型表现，定期更新

## ✅ 本章小结

- **机器学习** 让计算机从数据中自动学习规律
- 三大学习范式：**监督学习**（有标签）、**无监督学习**（无标签）、**强化学习**（奖励驱动）
- 核心概念：特征、标签、训练集/测试集、过拟合/欠拟合
- 从简单算法开始理解，逐步深入
- 机器学习项目有完整的工作流程：数据 → 训练 → 评估 → 部署
