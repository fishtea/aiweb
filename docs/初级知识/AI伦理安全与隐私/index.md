# AI 伦理安全与隐私

AI 系统会影响真实用户、业务流程和社会关系。初学阶段就应建立基本的安全、隐私和伦理意识，避免把模型输出当成绝对事实。

## 为什么重要

AI 风险不只来自技术错误，也来自使用方式：

- 错误建议可能影响医疗、金融、法律等高风险决策。
- 模型可能泄露或不当处理敏感信息。
- 训练数据偏见可能导致歧视性输出。
- 自动化系统可能放大错误，造成规模化影响。
- 生成内容可能被用于诈骗、钓鱼、虚假信息和侵权。

## 常见风险

| 风险 | 说明 | 缓解方式 |
|------|------|----------|
| 幻觉 | 生成不存在或错误的信息 | 引用来源、人工复核、RAG |
| 偏见 | 对群体产生不公平判断 | 数据审计、分组评估 |
| 隐私泄露 | 输入或输出包含敏感信息 | 脱敏、权限控制、日志治理 |
| 越权操作 | Agent 调用工具造成损害 | 最小权限、确认机制、审计 |
| 提示词注入 | 恶意文本操纵模型行为 | 指令分层、内容过滤、工具隔离 |
| 版权争议 | 生成内容可能近似已有作品 | 来源管理、授权检查、人工审核 |

## 敏感数据处理

不要随意把以下内容输入第三方模型：

- 身份证、护照、银行卡、手机号。
- 医疗记录、财务记录、合同细节。
- 未公开代码、密钥、token、内部文档。
- 用户对话、客户名单、商业机密。

如果业务确实需要处理敏感信息，应考虑脱敏、私有化部署、访问控制和合规审查。

## 人在回路

高风险场景应保留人工审核：

| 场景 | 建议 |
|------|------|
| 医疗建议 | AI 只能辅助整理信息，不能替代医生 |
| 法律合同 | AI 可做初稿和检查，最终需专业人士确认 |
| 金融风控 | 需要可解释、审计和人工申诉机制 |
| 招聘筛选 | 避免歧视性特征，做公平性评估 |
| 自动执行 Agent | 关键动作前要求用户确认 |

## 安全设计原则

- 最小权限：模型和工具只拿完成任务所需的权限。
- 可追溯：记录输入、输出、模型版本、工具调用和人工修改。
- 可回滚：重要自动化动作要能撤销。
- 可解释：关键决策要能说明依据。
- 可评估：上线前有测试集，上线后有监控。
- 可拒绝：模型应能在不确定或不合规时拒答。

## 初学者检查清单

- 这个任务是否涉及高风险决策？
- 输入中是否包含个人隐私或商业机密？
- 输出是否需要事实核验？
- 是否需要引用来源？
- 模型是否会调用外部工具或写入数据？
- 用户是否知道 AI 参与了处理？
- 错误发生后谁负责修正？
